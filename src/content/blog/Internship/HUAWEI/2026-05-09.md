## 第2题-5G基站设备故障风险预警（P4905）| 中等 | 1000ms

### 题目背景

华为的通信工程师需要对部署在户外的 5G 基站进行实时健康监控。现收集到一批基站的历史运行数据，包含两个关键指标：设备工作温度（特征 $x_1$）和天线负载系数（特征 $x_2$），以及该基站是否发生故障的标签（$y = 1$ 表示故障，$y = 0$ 表示正常）。

为了在边缘服务器上实现轻量级的实时预测，请你使用逻辑回归（Logistic Regression）结合批量梯度下降法（Batch Gradient Descent），从零开始训练一个故障预警模型，并预测新基站的故障概率。

### 模型公式与更新规则

**1. 预测函数**：对输入特征应用 sigmoid 激活函数：

$$z = w_1 x_1 + w_2 x_2 + b$$

$$\hat{y} = \frac{1}{1 + e^{-z}}$$

**2. 参数初始化**：所有权重和偏置初始值均设为 $0$（即 $w_1 = 0,\; w_2 = 0,\; b = 0$）。

**3. 梯度计算**：在每次迭代（Epoch）中，基于全体 N 个训练样本计算损失函数对各个参数的偏导数：

$$dw_1 = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i) \cdot x_{1i}$$

$$dw_2 = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i) \cdot x_{2i}$$

$$db = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i)$$

**4. 参数更新**：使用学习率 $\alpha$ 进行参数更新：

$$w_1 = w_1 - \alpha \cdot dw_1$$

$$w_2 = w_2 - \alpha \cdot dw_2$$

$$b = b - \alpha \cdot db$$

请严格按照上述步骤，迭代 $E$ 次后，输出给定新样本的预测故障概率。注：请统一使用 64 位双精度浮点数（double 或 float64）进行运算。

### 输入描述

所有数据通过标准输入读取，数值之间由一个或多个空格分隔。

- 第一行：三个数值，分别为训练样本数 $N$（整数）、迭代次数 $E$（整数）和学习率 $\alpha$（浮点数）
- 第 2 到第 $N+1$ 行：每行三个数值，分别为 $x_1$、$x_2$（浮点数）和标签 $y$（整数 0 或 1）
- 最后一行：两个数值，分别为待预测新样本的特征 $x_{1\_new}$ 和 $x_{2\_new}$（浮点数）

**参数范围约束**：
$1 \le N \le 1000$，$1 \le E \le 1000$，$0.001 \le \alpha \le 1.0$，$-100.0 \le x_1, x_2, x_{1\_new}, x_{2\_new} \le 100.0$

### 输出描述

输出一行，包含一个浮点数，表示新样本发生故障的预测概率。要求四舍五入保留到小数点后 4 位。

### 样例 1

**输入**

```
1 2 0.5
2.0 0.0 1
2.0 0.0
```

**输出**

```
0.8590
```

**说明**

初始化 $w_1 = 0,\; w_2 = 0,\; b = 0$。训练样本 1 个 ($x_1=2.0,\; x_2=0.0,\; y=1$)，迭代 2 次，学习率 0.5。第 1 次迭代：$z = 0$，$\hat{y} = 0.5$，误差 = $0.5 - 1 = -0.5$，梯度更新得到 $w_1 = 0.5,\; w_2 = 0,\; b = 0.25$。第 2 次迭代后预测新样本概率为 0.8590。

### 样例 2

**输入**

```
2 1 0.1
1.0 2.0 1
-1.0 -2.0 0
0.5 1.0
```

**输出**

```
0.5312
```

**说明**

初始化：$w_1 = 0,\; w_2 = 0,\; b = 0$。第 1 次迭代：
- 样本 1 ($y=1$)：$z_1 = 0$ → $\hat{y}_1 = 0.5$，误差 $= 0.5 - 1 = -0.5$
- 样本 2 ($y=0$)：$z_2 = 0$ → $\hat{y}_2 = 0.5$，误差 $= 0.5 - 0 = 0.5$

梯度计算：
- $dw_1 = ((-0.5 \times 1.0) + (0.5 \times -1.0)) / 2 = -0.5$
- $dw_2 = ((-0.5 \times 2.0) + (0.5 \times -2.0)) / 2 = -1.0$
- $db = (-0.5 + 0.5) / 2 = 0.0$

参数更新：
- $w_1 = 0 - 0.1 \times (-0.5) = 0.05$
- $w_2 = 0 - 0.1 \times (-1.0) = 0.1$
- $b = 0 - 0.1 \times 0.0 = 0.0$

预测新样本：$z = 0.05 \times 0.5 + 0.1 \times 1.0 + 0 = 0.125$

$Probability = 1 / (1 + e^{-0.125}) \approx 0.5312$

## 第3题-多头注意力掩码计算（P4906）| 中等 | 2000ms

### 问题背景

在大模型的训练和推理中，为了提高 GPU 的效率，通常将多个不同长度的句子打包成一个 Batch：

- **Batch（批处理）**：不同句子长度不一，短句子后面会补零（Padding）
- **Multi-Head（多头）**：向量维度会被拆分成多个"头"，每个头独立计算注意力，以捕捉不同层面的语义
- **双重掩码**：在计算时必须同时满足因果掩码（不看未来）和 Padding 掩码（不看填充位置）
- **位置惩罚**：对未被掩码的位置进行偏置惩罚，惩罚因子随头的索引而变化（避免所有头关注范围重合）。惩罚因子 $slope = s + 1$，其中 $s$ 为头索引（从 0 开始）

### 主要步骤

给定一个 Batch 的注意力得分张量（已经过缩放 $Score = QK^T / \sqrt{d_k}$），请针对每批次和头，实现其掩码注入和归一化过程。

**步骤 1：因果掩码**

对于一个给定的 $n \times n$ 得分矩阵 $Score$，其上三角位置（不包括对角线）的值替换为 $-1e9$，其他位置不变，防止模型看到"未来"的信息。

**步骤 2：Padding 掩码注入**

给定每个序列的有效长度 $L$，若列索引 $j \ge L$（列索引从 0 开始），则 $Score_{i,j} = -1e9$（即该 Token 是填充的，不应被关注）。

**步骤 3：位置惩罚**

注入距离偏置，对未被掩码的位置（即下三角位置），在其原始得分的基础上减去距离偏置：

- 偏置计算：$Bias_{i,j} = (i - j) \times slope$
- 更新公式：$Score'_{i,j} = Score_{i,j} - Bias_{i,j}$

**步骤 4：归一化（Safe Softmax）**

对处理后的矩阵每一行进行 Safe Softmax 归一化：

对于每一行 $i$，先找到该行中非掩码位置集合 $valid_i$，然后找到该行的最大值 $M_i$。

对于行 $i$ 列 $j$ 的元素：
- 如果 $j \notin valid_i$（即 $Score'_{i,j} = -1e9$），则 $Softmax(Score'_{i,j}) = 0$
- 如果 $j \in valid_i$，则：

$$Softmax(Score'_{i,j}) = \frac{e^{Score'_{i,j} - M_i}}{\sum_{j \in valid_i} e^{Score'_{i,j} - M_i}}$$

### 输入格式

- 第一行：三个整数 $B$（批数量）、$H$（"头"数量）、$S$（得分矩阵维度）
- 第二行：$B$ 个整数，代表每个 Batch 的有效序列长度 $L_b$（最小值为 0）
- 接下来 $B \times H \times S$ 行，每行 $S$ 个浮点数，数据按 $B \to H \to S$ 顺序排列，是经过缩放之后的得分矩阵 $Score$

### 输出格式

输出处理后的张量，每行 $S$ 个浮点数，保留 4 位小数。
