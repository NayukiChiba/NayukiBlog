## 第1题-选择题（P4904）共20道，1-15单选，16-20多选

### 1. 逻辑回归的决策边界是？
- A. S形曲线
- B. 特征空间中的超平面 $w^T x + b = 0$
- C. 以原点为圆心的超球面
- D. 由核函数决定的非线性曲面

:::fold[答案]

正确答案：B

解析：逻辑回归的决策边界是线性超平面 $w^T x + b = 0$，模型通过 Sigmoid 函数将线性输出映射到概率，但决策边界本身是线性的。
:::

### 2. CUDA内核寄存器溢出时数据存储到哪种内存？
- A. L1缓存
- B. 局部内存（由全局内存物理支持）
- C. 常量内存
- D. 共享内存

:::fold[答案]

正确答案：B

解析：CUDA 寄存器溢出时，变量会被存储到局部内存（Local Memory），该内存在物理上位于全局内存（Global Memory/DRAM），访问延迟高。L1 缓存、共享内存和常量内存均由编译器显式管理，不用于寄存器溢出。
:::

### 3. 固定页管理KV-Cache在显存碎片严重场景下最直接的好处？
- A. 减少小对象频繁分配导致的碎片与分配开销
- B. 减少碎片对不齐提高数值精度
- C. 降低显存访问带宽压力
- D. 降低整体显存使用量

:::fold[答案]

正确答案：A

解析：固定页（Block）管理 KV Cache（如 vLLM 的 PagedAttention）通过将 KV Cache 划分为固定大小的页来分配内存，最直接的好处是减少小对象频繁分配/释放导致的显存碎片和分配开销，提高显存利用率。
:::

### 4. 长文本生成模型中关于温度系数，正确的是？
- 选项见原题

:::fold[答案]

正确答案：温度系数 $T$ 控制输出概率分布的平滑程度，$T > 1$ 使分布更均匀（增加随机性/多样性），$T < 1$ 使分布更尖锐（更确定性），$T \to 0$ 趋近于贪婪搜索。
:::

### 5. 关于Decoder-only，错误的说法是？
- C. 必须包含独立编码器才能生成输出

:::fold[答案]

正确答案：C

解析：Decoder-only 架构（如 GPT 系列）仅使用解码器，不包含独立的编码器，通过因果自注意力掩码实现自回归生成。"必须包含独立编码器"是错误说法。
:::

### 6. PPO中截断代理目标函数的主要目的是？
- D. 限制策略更新幅度，防止破坏性的大幅更新

:::fold[答案]

正确答案：D

解析：PPO 的截断代理目标函数通过 clip 操作限制新旧策略的概率比在 $[1-\epsilon, 1+\epsilon]$ 范围内，防止单步更新幅度过大导致策略崩溃，是 PPO 稳定训练的核心机制。
:::

### 7. Prompt Tuning（Prefix Tuning）的正确描述？
- D. 在输入前添加可学习的连续向量，仅更新这些向量参数

:::fold[答案]

正确答案：D

解析：Prompt Tuning/Prefix Tuning 的核心思想是在输入序列前拼接可学习的连续向量（soft prompts），训练时冻结预训练模型参数，仅优化这些新增的连续向量，实现参数高效微调。
:::

### 8. 梯度爆炸问题通常与权重矩阵的什么性质相关？
- A. 特征值过大

:::fold[答案]

正确答案：A

解析：梯度爆炸的根源在于权重矩阵的最大特征值（或谱半径）$|\lambda_{\max}| > 1$，导致反向传播时梯度随层数指数增长。若特征值均小于 1 则可能导致梯度消失。
:::

### 9. 二项分布 $B(m,p)$ 中 $p$ 的最大似然估计值为？
- 样本均值/m

:::fold[答案]

正确答案：$\hat{p} = \frac{\bar{X}}{m}$

解析：二项分布 $X \sim B(m, p)$ 中，$m$ 次独立伯努利试验的 MLE 为 $\hat{p} = \frac{\sum_{i=1}^{n} X_i}{n \cdot m} = \frac{\bar{X}}{m}$，即样本均值除以试验次数。
:::

### 10. 自注意力 $O(N^2)$ 瓶颈——扩展上下文窗口显存增长倍数？
- 选项见原题

:::fold[答案]

正确答案：序列长度从 $N$ 扩展到 $2N$ 时，注意力矩阵大小从 $N \times N$ 变为 $2N \times 2N$，显存增长约 4 倍（$O(N^2)$ 性质），这是标准自注意力的核心瓶颈。
:::

### 11. Scaled Dot-Product Attention除以 $\sqrt{d_k}$ 的原因？
- 防止点积过大导致softmax梯度消失

:::fold[答案]

正确答案：防止点积过大导致 Softmax 梯度消失

解析：$QK^T$ 的方差随 $d_k$ 增大而线性增长，除以 $\sqrt{d_k}$ 将方差归一化到 1，避免点积值过大使 Softmax 进入梯度饱和区（梯度趋近于 0），保证训练稳定。
:::

### 12. PCA降维中协方差矩阵与投影矩阵的维度分析
- 见原题

:::fold[答案]

正确答案：对于 $n$ 个样本、$d$ 个特征的数据集，协方差矩阵维度为 $d \times d$；选取前 $k$ 个主成分时，投影矩阵维度为 $k \times d$（或 $d \times k$，取决于乘法顺序）。
:::

### 13. 逻辑回归sigmoid：斜率1.5，截距-15，时长10分钟，潜在用户概率？
- 见原题

:::fold[答案]

正确答案：概率为 50%（0.5）

解析：$z = 1.5 \times 10 + (-15) = 0$，Sigmoid 函数 $\sigma(0) = \frac{1}{1+e^0} = 0.5$，即潜在用户概率为 50%。
:::

### 14. SFT后模型出现"过度道歉"现象最可能原因？
- SFT训练数据中过度包含拒绝回答/道歉类样本

:::fold[答案]

正确答案：SFT 训练数据中过度包含拒绝回答/道歉类样本

解析：SFT 阶段模型直接模仿训练数据的回答风格，若数据中包含大量"对不起，我不能..."或过度安全的回复模式，模型会习得这种"过度道歉"的行为模式。
:::

### 15. 贝叶斯网络 $A \to B \to C$，给定 $B$ 时 $A$ 与 $C$ 的关系？
- B. A与C条件独立

:::fold[答案]

正确答案：B

解析：在贝叶斯网络 $A \to B \to C$（链式结构/因果链）中，$B$ 是 $A$ 和 $C$ 的中间节点。给定 $B$ 后，$A$ 与 $C$ 被 $d$-分离，满足条件独立，即 $A \perp C \mid B$。
:::

### 16.（多选）KV缓存机制在大模型推理中的作用

:::fold[答案]

正确答案：KV Cache 缓存历史 token 的 Key 和 Value 向量，避免自回归生成时每步重复计算所有历史 token 的 K/V；显著减少 Decode 阶段计算量；以显存换时间，使自回归推理从 $O(N^2)$ 降至 $O(N)$ 每步。
:::

### 17.（多选）数据集{3,4,5,3,8,7}的统计描述

:::

### 18.（多选）工业级ML管道中特征处理与选择的深层逻辑

:::fold[答案]

正确答案：特征工程应考虑特征间交互而非仅独立评估；缺失值处理策略（均值填充/模型预测/删除）需根据缺失机制（MCAR/MAR/MNAR）选择；特征选择应结合过滤法（相关性筛选）和嵌入法（如 L1 正则化/LightGBM 特征重要性），平衡计算效率与模型性能。

### 19.（多选）INT8量化部署中影响模型精度的因素

:::fold[答案]

正确答案：量化粒度（逐层/逐通道）越细精度越高但开销越大；校准数据集的选择影响激活值量化范围的统计准确性；异常值（outlier）会扩大量化范围导致精度显著下降；KV Cache 量化比权重量化对精度更敏感；逐层量化误差会随层数累积。

:::fold[答案]

:::

## 第2题-5G基站设备故障风险预警（P4905）| 中等 | 1000ms

### 题目背景

华为的通信工程师需要对部署在户外的 5G 基站进行实时健康监控。现收集到一批基站的历史运行数据，包含两个关键指标：设备工作温度（特征 $x_1$）和天线负载系数（特征 $x_2$），以及该基站是否发生故障的标签（$y = 1$ 表示故障，$y = 0$ 表示正常）。

为了在边缘服务器上实现轻量级的实时预测，请你使用逻辑回归（Logistic Regression）结合批量梯度下降法（Batch Gradient Descent），从零开始训练一个故障预警模型，并预测新基站的故障概率。

### 模型公式与更新规则

**1. 预测函数**：对输入特征应用 sigmoid 激活函数：

$$
z = w_1 x_1 + w_2 x_2 + b
$$

$$
\hat{y} = \frac{1}{1 + e^{-z}}
$$

**2. 参数初始化**：所有权重和偏置初始值均设为 $0$（即 $w_1 = 0,\; w_2 = 0,\; b = 0$）。

**3. 梯度计算**：在每次迭代（Epoch）中，基于全体 N 个训练样本计算损失函数对各个参数的偏导数：

$$
dw_1 = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i) \cdot x_{1i}
$$

$$
dw_2 = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i) \cdot x_{2i}
$$

$$
db = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i)
$$

**4. 参数更新**：使用学习率 $\alpha$ 进行参数更新：

$$
w_1 = w_1 - \alpha \cdot dw_1
$$

$$
w_2 = w_2 - \alpha \cdot dw_2
$$

$$
b = b - \alpha \cdot db
$$

请严格按照上述步骤，迭代 $E$ 次后，输出给定新样本的预测故障概率。注：请统一使用 64 位双精度浮点数（double 或 float64）进行运算。

### 输入描述

所有数据通过标准输入读取，数值之间由一个或多个空格分隔。

- 第一行：三个数值，分别为训练样本数 $N$（整数）、迭代次数 $E$（整数）和学习率 $\alpha$（浮点数）
- 第 2 到第 $N+1$ 行：每行三个数值，分别为 $x_1$、$x_2$（浮点数）和标签 $y$（整数 0 或 1）
- 最后一行：两个数值，分别为待预测新样本的特征 $x_{1\_new}$ 和 $x_{2\_new}$（浮点数）

**参数范围约束**：
$1 \le N \le 1000$，$1 \le E \le 1000$，$0.001 \le \alpha \le 1.0$，$-100.0 \le x_1, x_2, x_{1\_new}, x_{2\_new} \le 100.0$

### 输出描述

输出一行，包含一个浮点数，表示新样本发生故障的预测概率。要求四舍五入保留到小数点后 4 位。

### 样例 1

**输入**

```
1 2 0.5
2.0 0.0 1
2.0 0.0
```

**输出**

```
0.8590
```

**说明**

初始化 $w_1 = 0,\; w_2 = 0,\; b = 0$。训练样本 1 个 ($x_1=2.0,\; x_2=0.0,\; y=1$)，迭代 2 次，学习率 0.5。第 1 次迭代：$z = 0$，$\hat{y} = 0.5$，误差 = $0.5 - 1 = -0.5$，梯度更新得到 $w_1 = 0.5,\; w_2 = 0,\; b = 0.25$。第 2 次迭代后预测新样本概率为 0.8590。

### 样例 2

**输入**

```
2 1 0.1
1.0 2.0 1
-1.0 -2.0 0
0.5 1.0
```

**输出**

```
0.5312
```

**说明**

初始化：$w_1 = 0,\; w_2 = 0,\; b = 0$。第 1 次迭代：
- 样本 1 ($y=1$)：$z_1 = 0$ → $\hat{y}_1 = 0.5$，误差 $= 0.5 - 1 = -0.5$
- 样本 2 ($y=0$)：$z_2 = 0$ → $\hat{y}_2 = 0.5$，误差 $= 0.5 - 0 = 0.5$

梯度计算：
- $dw_1 = ((-0.5 \times 1.0) + (0.5 \times -1.0)) / 2 = -0.5$
- $dw_2 = ((-0.5 \times 2.0) + (0.5 \times -2.0)) / 2 = -1.0$
- $db = (-0.5 + 0.5) / 2 = 0.0$

参数更新：
- $w_1 = 0 - 0.1 \times (-0.5) = 0.05$
- $w_2 = 0 - 0.1 \times (-1.0) = 0.1$
- $b = 0 - 0.1 \times 0.0 = 0.0$

预测新样本：$z = 0.05 \times 0.5 + 0.1 \times 1.0 + 0 = 0.125$

$Probability = 1 / (1 + e^{-0.125}) \approx 0.5312$

## 解题思路
使用逻辑回归的批量梯度下降法。初始化 $w_1 = w_2 = b = 0$，每轮迭代先计算所有样本的 sigmoid 预测值，再求损失函数对各参数的偏导数（均值），最后用学习率更新参数。$E$ 轮迭代后，用训练好的参数对新样本做 sigmoid 预测得到故障概率。全程使用 float64 精度，输出四舍五入到4位小数。

## 参考代码
```python
import sys
import math

def sigmoid(z):
    """sigmoid函数，带溢出保护"""
    if z > 50:
        return 1.0
    if z < -50:
        return 0.0
    return 1.0 / (1.0 + math.exp(-z))

def solve():
    data = sys.stdin.read().strip().split()
    it = iter(data)
    N = int(next(it))
    E = int(next(it))
    alpha = float(next(it))

    samples = []
    for _ in range(N):
        x1 = float(next(it))
        x2 = float(next(it))
        y = int(next(it))
        samples.append((x1, x2, y))

    x1New = float(next(it))
    x2New = float(next(it))

    # 参数初始化
    w1 = 0.0
    w2 = 0.0
    b = 0.0

    # 批量梯度下降
    for epoch in range(E):
        # 计算所有样本的预测值
        preds = []
        for x1, x2, y in samples:
            z = w1 * x1 + w2 * x2 + b
            preds.append(sigmoid(z))

        # 计算梯度
        dw1 = 0.0
        dw2 = 0.0
        db = 0.0
        for i, (x1, x2, y) in enumerate(samples):
            error = preds[i] - y
            dw1 += error * x1
            dw2 += error * x2
            db += error

        dw1 /= N
        dw2 /= N
        db /= N

        # 参数更新
        w1 -= alpha * dw1
        w2 -= alpha * dw2
        b -= alpha * db

    # 预测新样本
    zNew = w1 * x1New + w2 * x2New + b
    prob = sigmoid(zNew)
    print(f'{prob:.4f}')

if __name__ == '__main__':
    solve()
```

## 第3题-多头注意力掩码计算（P4906）| 中等 | 2000ms

### 问题背景

在大模型的训练和推理中，为了提高 GPU 的效率，通常将多个不同长度的句子打包成一个 Batch：

- **Batch（批处理）**：不同句子长度不一，短句子后面会补零（Padding）
- **Multi-Head（多头）**：向量维度会被拆分成多个"头"，每个头独立计算注意力，以捕捉不同层面的语义
- **双重掩码**：在计算时必须同时满足因果掩码（不看未来）和 Padding 掩码（不看填充位置）
- **位置惩罚**：对未被掩码的位置进行偏置惩罚，惩罚因子随头的索引而变化（避免所有头关注范围重合）。惩罚因子 $slope = s + 1$，其中 $s$ 为头索引（从 0 开始）

### 主要步骤

给定一个 Batch 的注意力得分张量（已经过缩放 $Score = QK^T / \sqrt{d_k}$），请针对每批次和头，实现其掩码注入和归一化过程。

**步骤 1：因果掩码**

对于一个给定的 $n \times n$ 得分矩阵 $Score$，其上三角位置（不包括对角线）的值替换为 $-1e9$，其他位置不变，防止模型看到"未来"的信息。

**步骤 2：Padding 掩码注入**

给定每个序列的有效长度 $L$，若列索引 $j \ge L$（列索引从 0 开始），则 $Score_{i,j} = -1e9$（即该 Token 是填充的，不应被关注）。

**步骤 3：位置惩罚**

注入距离偏置，对未被掩码的位置（即下三角位置），在其原始得分的基础上减去距离偏置：

- 偏置计算：$Bias_{i,j} = (i - j) \times slope$
- 更新公式：$Score'_{i,j} = Score_{i,j} - Bias_{i,j}$

**步骤 4：归一化（Safe Softmax）**

对处理后的矩阵每一行进行 Safe Softmax 归一化：

对于每一行 $i$，先找到该行中非掩码位置集合 $valid_i$，然后找到该行的最大值 $M_i$。

对于行 $i$ 列 $j$ 的元素：
- 如果 $j \notin valid_i$（即 $Score'_{i,j} = -1e9$），则 $Softmax(Score'_{i,j}) = 0$
- 如果 $j \in valid_i$，则：

$$
Softmax(Score'_{i,j}) = \frac{e^{Score'_{i,j} - M_i}}{\sum_{j \in valid_i} e^{Score'_{i,j} - M_i}}
$$

### 输入格式

- 第一行：三个整数 $B$（批数量）、$H$（"头"数量）、$S$（得分矩阵维度）
- 第二行：$B$ 个整数，代表每个 Batch 的有效序列长度 $L_b$（最小值为 0）
- 接下来 $B \times H \times S$ 行，每行 $S$ 个浮点数，数据按 $B \to H \to S$ 顺序排列，是经过缩放之后的得分矩阵 $Score$

### 输出格式

输出处理后的张量，每行 $S$ 个浮点数，保留 4 位小数。

0.1192 0.8808 0.0000
0.0159 0.1173 0.8668

0.0000 0.0000 0.0000
1.0000 0.0000 0.0000

**说明**

$B=2$（2 个 batch），$H=2$（2 个头），$S=3$（得分矩阵 $3 \times 3$），共 4 个矩阵。Batch 1 有效长度 3，Batch 2 有效长度 2。头 0 的 $slope = 1$，头 1 的 $slope = 2$。因果掩码将上三角置为 $-1e9$，Padding 掩码将 batch 2 的列 2 置为 $-1e9$，位置惩罚对下三角减去距离偏置，最后每行做 Safe Softmax。

### 样例 2

**输入**

```
2 2 2
2 1
10.0 10.0
10.0 10.0
10.0 10.0
10.0 10.0
5.0 5.0
5.0 5.0
5.0 5.0
5.0 5.0
```

**输出**

```
1.0000 0.0000
0.2689 0.7311
1.0000 0.0000
0.1192 0.8808
1.0000 0.0000
0.0000 0.0000
1.0000 0.0000
0.0000 0.0000
```

**说明**

$B=2$，$H=2$，$S=2$（得分矩阵 $2 \times 2$），共 4 个矩阵。Batch 1 有效长度 2，Batch 2 有效长度 1。Batch 2 由于有效长度仅为 1，列索引 $\ge 1$ 的位置全部被 Padding 掩码置为 $-1e9$，其第二行全部被掩码，Softmax 结果为全 0。

## 解题思路
对每个batch的每个头，分四步处理：步骤1将得分矩阵上三角（j>i）置为-1e9实现因果掩码；步骤2将列索引≥有效长度L的位置置为-1e9实现Padding掩码；步骤3对未被掩码的下三角位置减去距离偏置 $(i-j) \cdot \text{slope}$（$\text{slope} = \text{头索引} + 1$）；步骤4对每行做 Safe Softmax（先求行最大值减去后 exp 求和，掩码位置输出 0）。所有输出保留4位小数。

## 参考代码
```python
import sys
import math

def solve():
    data = sys.stdin.read().strip().split()
    it = iter(data)
    B = int(next(it))
    H = int(next(it))
    S = int(next(it))

    # 有效序列长度
    seqLens = [int(next(it)) for _ in range(B)]

    # 读取得分矩阵：B × H 个矩阵，每个 S × S
    # 顺序：先按 batch，再按 head
    scores = []  # scores[b][h][i][j]
    for b in range(B):
        batchScores = []
        for h in range(H):
            matrix = []
            for i in range(S):
                row = [float(next(it)) for _ in range(S)]
                matrix.append(row)
            batchScores.append(matrix)
        scores.append(batchScores)

    NEG_INF = -1e9

    # 处理每个 batch 的每个 head
    for b in range(B):
        L = seqLens[b]
        for h in range(H):
            matrix = scores[b][h]
            slope = h + 1  # 惩罚因子 = 头索引 + 1

            # 步骤1：因果掩码（上三角不包括对角线）
            for i in range(S):
                for j in range(i + 1, S):
                    matrix[i][j] = NEG_INF

            # 步骤2：Padding掩码
            for i in range(S):
                for j in range(L, S):
                    matrix[i][j] = NEG_INF

            # 步骤3：位置惩罚（仅对未被掩码的位置）
            for i in range(S):
                for j in range(i + 1):  # 下三角（含对角线）
                    if matrix[i][j] > NEG_INF / 2:  # 未被掩码
                        bias = (i - j) * slope
                        matrix[i][j] -= bias

            # 步骤4：Safe Softmax（按行）
            for i in range(S):
                # 找出该行有效位置和最大值
                valid = [j for j in range(S) if matrix[i][j] > NEG_INF / 2]
                if not valid:
                    # 整行被掩码，全输出0
                    for j in range(S):
                        matrix[i][j] = 0.0
                    continue

                maxVal = max(matrix[i][j] for j in valid)

                # 计算exp并求和
                expSum = 0.0
                expVals = {}
                for j in valid:
                    expVals[j] = math.exp(matrix[i][j] - maxVal)
                    expSum += expVals[j]

                # 归一化
                for j in range(S):
                    if j in expVals:
                        matrix[i][j] = expVals[j] / expSum
                    else:
                        matrix[i][j] = 0.0

    # 输出：按 B -> H -> S 顺序，每行S个浮点数
    for b in range(B):
        for h in range(H):
            for i in range(S):
                row = scores[b][h][i]
                print(' '.join(f'{v:.4f}' for v in row))

if __name__ == '__main__':
    solve()
```
