### 1. Swish 激活函数定义为 $f(x) = x \cdot \text{sigmoid}(x)$。相比于 ReLU，Swish 的主要特点是？

- A. 它是非单调的
- B. 它是完全非负的
- C. 它的计算成本比 ReLU 低得多
- D. 它在负区间有平滑的曲线，可能保留更多信息

:::fold[答案]

正确答案：A

解析：Swish 是少数非单调的激活函数之一，在 $x < 0$ 区域存在微小的负值下探，这是其相比 ReLU 最本质的数学特征。D 描述的平滑性在 GELU、ELU 等中也存在，非单调性才是 Swish 的标志性特点。
:::

### 2. 在使用 ZeRO-1 优化的数据并行中，每个 GPU 计算完局部梯度后，通过哪两个基础通信算子以实现梯度的聚合和分片？

- A. All-Gather 和 Broadcast
- B. Broadcast 和 Reduce
- C. Scatter 和 All-Reduce
- D. Reduce-Scatter 和 All-Gather

:::fold[答案]

正确答案：D

解析：ZeRO-1 中，各 GPU 先通过 Reduce-Scatter 将梯度按参数量分片聚合到对应 GPU，参数更新后再通过 All-Gather 将更新后的完整参数广播到所有 GPU。
:::

### 3. 在表格数据处理中，若某列包含连续数值，另一列包含类别标签（如"男/女"），为了输入神经网络，通常的做法是？

- A. 全部归一化为 0-1 之间
- B. 连续数值归一化，类别标签 One-Hot 编码
- C. 全部转换为字符串
- D. 连续数值 One-Hot 编码，类别标签归一化

正确答案：B

### 4. 大模型 INT8/FP8 量化时，若设定截断阈值 $T = 3\sigma$，基于高斯分布性质，"$3\sigma$ 截断法则"会导致大约多少比例的原始数据产生截断失真？

- C. 约 $31.7\%$
- D. 约 $0.27\%$

:::fold[答案]

正确答案：D

### 5. 在构建线性回归模型时，发现训练误差很小，但测试误差很大，最可能的原因是

- A. 欠拟合
- B. 过拟合

### 6. 关于协方差与独立性的命题，正确的一项是：

- A. 若 $D(X+Y) = D(X) + D(Y)$ 成立，则 $X$ 和 $Y$ 一定不相关
- B. 若协方差为 0（不相关），则一定相互独立
- C. 若相互独立，则协方差 $\text{Cov}(X,Y)$ 可能不为 0
- D. 若服从二维正态分布且不相关，则不一定相互独立

:::fold[答案]

正确答案：A

解析：$D(X+Y) = D(X) + D(Y) + 2\text{Cov}(X,Y)$，等式成立意味着 $\text{Cov}(X,Y) = 0$，即不相关。B 错误，不相关不一定独立（反例：$Y = X^2$ 在对称分布下）；C 错误，独立必然协方差为 0；D 错误，二维正态下不相关与独立等价。
:::

### 8. 已知对称矩阵 $A = \begin{bmatrix} 2 & 1 \\ 1 & 2 \end{bmatrix}$，求解特征值和特征向量

- A. $\lambda_1 = 1,\ \nu_1 = [1, 1];\ \lambda_2 = 3,\ \nu_2 = [1, -1]$
- B. $\lambda_1 = 2,\ \nu_1 = [1, 0];\ \lambda_2 = 1,\ \nu_2 = [0, 1]$
- C. $\lambda_1 = 3,\ \nu_1 = [1, 1];\ \lambda_2 = 1,\ \nu_2 = [1, -1]$
- D. $\lambda_1 = 4,\ \nu_1 = [1, 1];\ \lambda_2 = 0,\ \nu_2 = [1, -1]$

:::fold[答案]

正确答案：C

解析：特征方程 $\det(A - \lambda I) = (2-\lambda)^2 - 1 = \lambda^2 - 4\lambda + 3 = 0$，解得 $\lambda_1 = 3,\ \lambda_2 = 1$。$\lambda = 3$ 时 $[1, 1]^T$ 是特征向量；$\lambda = 1$ 时 $[1, -1]^T$ 是特征向量。
:::

### 10. 包含异常值较多的数据集，使用对距离敏感的算法（SVM/KNN），哪种特征缩放最鲁棒？

- A. Standardization（标准化）

:::fold[答案]

正确答案：B

:::

### 11. 模型用 FP8 训练，工程上常见的数值稳定化操作是：

- A. 梯度缩放
- B. 去掉 dropout
- C. 使用更小的 batch size，避免过度平滑
- D. 增大学习率并减少权重衰减

:::fold[答案]

正确答案：A

解析：FP8 的动态范围极小（指数位少），梯度值容易下溢为 0。梯度缩放（Gradient Scaling / Loss Scaling）将损失放大后再反向传播，最后缩小梯度，可有效防止 FP8 低精度下的梯度下溢。
:::

### 12. 空洞卷积（Dilated Convolution）的作用是

- A. 增加感受野而不增加参数
- B. 提高特征图分辨率
- C. 以上都不对
- D. 减少计算量

:::fold[答案]

正确答案：A

解析：空洞卷积通过在卷积核元素之间插入空洞（间隔）来扩大感受野，而不增加额外参数或计算量。它不提高特征图分辨率（反而可能降低），计算量与同核大小的标准卷积相当。

### 13. 多服务器集群，单机 8 卡 NVLink，跨机 InfiniBand。最合理的并行策略映射是？

- B. PP 在单机 8 卡内，TP 跨节点
- C. DP 在单机 8 卡内，TP 和 PP 跨节点

### 14. 如果 $\text{Cov}(X,Y) = 10$, $\text{Var}(X) = 4$, $\text{Var}(Y) = 25$，则 $X$ 和 $Y$ 的相关系数是？

- B. $0.8$

### 15. 函数 $f(x) = \frac{1}{1 + 25x^2}$ 在 $[-1, 1]$ 上取等距节点高次插值，当 $n$ 增大时？

### 16.（多选）SmoothQuant 引入逐通道缩放向量 $s$ 进行变换：以下说法正确的有？

- A. 若某通道有离群值，选择 $s_j$ 为该通道的 $\ell_{\infty}$ 范数，可将该通道压缩至 $[-1, 1]$，显著降低量化难度
- B. 变换后权重行范数会随 $s_j$ 同比增大，需在"激活易量化"与"权重量化"间平衡
- C. 该变换只能用于全连接层和卷积层，无法用于 Q/K/V 投影
- D. 该变换数学上严格等价，对任意 $s_i > 0$ 浮点精度下结果完全一致

:::fold[答案]

正确答案：A, B, D

解析：SmoothQuant 的核心是 $Y = XW = (X\text{diag}(s)^{-1})(\text{diag}(s)W)$，数学上严格等价（D）。$s_j = \|X_{:,j}\|_{\infty}$ 可将离群通道压缩到 $[-1,1]$（A）。权重行 $i$ 被缩放 $s_i$ 倍，需权衡激活和权重的量化难度（B）。C 错误，该变换适用于所有线性层包括 Q/K/V 投影。
:::

## 第2题-大模型流水线并行训练优化（P4982）| 困难 | 2000ms

### 题目描述

在华为昇腾（Ascend）集群上训练千亿参数的大模型时，由于单个昇腾 NPU 的 HBM 显存无法装下整个模型，通常采用**流水线并行（Pipeline Parallelism）**技术。该技术将模型按序切分为多个连续的阶段（Stage），每个阶段分配到集群中不同的 NPU 上。

现有一个包含 N 层的神经网络模型，需要按顺序切分并部署到 K 个昇腾 NPU 上，即划分为 K 个连续的流水线阶段。每个 NPU 负责一个阶段，且每个 NPU **至少需要分配 1 层**网络。

已知：

- 第 i 层网络前向与反向传播的计算耗时为 **C[i]**
- 第 i 层网络所需的显存容量为 **W[i]**
- 每个昇腾 NPU 的显存物理上限为 **M**

在流水线并行中，整个集群的吞吐效率受限于最慢的那个 NPU（即计算耗时最大的阶段，称为流水线瓶颈）。一个阶段的计算耗时等于分配给它的所有网络层 C[i] 的总和，其所需显存等于这些网络层 W[i] 的总和。

如果任意一个阶段的显存总和超过了单卡上限 M，则会导致 OOM（内存溢出），该切分方案无效。

**注意**：由于神经网络计算图的数据流向依赖，分配给同一个 NPU 的多个网络层必须在原始顺序中是连续的，不可打乱或跨层自由组合。

**任务**：设计一个算法，在保证不发生 OOM 的前提下，找到一种切分方案，使得所有 NPU 中最大的计算耗时尽可能小。输出这个最小的"最大计算耗时"。如果无法找到满足显存限制的切分方案，或者 NPU 数量多于网络层数（无法保证每卡至少 1 层），输出 **-1**。

### 输入描述

输入共三行，所有数字均通过空格分隔。

第一行：三个正整数 `N K M`，分别代表网络总层数、NPU 数量（阶段数）、单卡显存上限。

第二行：N 个正整数，代表每一层的计算耗时数组 C。

第三行：N 个正整数，代表每一层的显存需求数组 W。

### 输出描述

一个整数，表示在满足显存约束下最小化的最大计算耗时。如果无解，输出 -1。

### 样例1

**输入**
```
3 4 100
1 1 1
1 1 1
```

**输出**
```
-1
```

**说明**

NPU 数量（4）大于模型层数（3）。由于必须保证每个 NPU 至少分配一层网络，无法满足条件，返回 -1。

### 样例2

**输入**
```
4 2 10
2 2 2 2
6 6 6 6
```

**输出**
```
-1
```

**说明**

单卡显存上限为 10，但每层都需要 6 的显存。如果将任意两层放在同一个 NPU 上，显存需求就是 12，会发生 OOM。因此 4 层网络必须每层单独占用一个 NPU（需要 4 个 NPU），但集群只有 2 个 NPU，故无法完成切分，返回 -1。

### 样例3

**输入**
```
5 3 20
5 1 2 3 4
10 5 5 5 10
```

**输出**
```
6
```

**说明**

模型共 5 层，需分配到 3 个 NPU，单卡显存上限 20。

最佳切分方案：[层0, 层1] → NPU1, [层2, 层3] → NPU2, [层4] → NPU3。

| NPU | 分配层 | 计算耗时 | 显存 | 是否合法 |
|-----|-------|---------|------|---------|
| NPU1 | [0, 1] | 5+1=6 | 10+5=15 ≤ 20 | 合法 |
| NPU2 | [2, 3] | 2+3=5 | 5+5=10 ≤ 20 | 合法 |
| NPU3 | [4] | 4 | 10 ≤ 20 | 合法 |

最大计算耗时为 6，没有比 6 更优的合法方案。

### 数据约束与算法提示

- 核心算法：**二分答案 + 贪心验证**
- 二分搜索最大计算耗时 mid，贪心地将层分配给当前 NPU（计算耗时累加不超过 mid 且显存不超 M），检查 K 个 NPU 内能否分配完所有层
- 注意事项：NPU 数 > 层数 → 直接返回 -1；任意单层显存 > M → 直接返回 -1

:::fold[答案]
## 解题思路
二分答案 + 贪心验证。二分搜索最大计算耗时上限 mid，贪心地将连续层分配给当前 NPU：累加 C[i] 不超过 mid 且累加 W[i] 不超过 M，超出任一限制则新开一个 NPU。若能以 ≤ K 个 NPU 完成分配则 mid 可行，缩小上界；否则增大下界。边界条件：K > N 或存在单层 W[i] > M 直接返回 -1。

## 参考代码
```python
import sys

def canPartition(layers, K, M, maxCost):
    """检查是否能让每个 NPU 的计算耗时 <= maxCost 且显存 <= M"""
    groups = 0
    i = 0
    n = len(layers)
    while i < n:
        curCost = 0
        curMem = 0
        # 当前 NPU 尽可能多地放层
        while i < n:
            c, w = layers[i]
            if curCost + c <= maxCost and curMem + w <= M:
                curCost += c
                curMem += w
                i += 1
            else:
                break
        groups += 1
        if groups > K:
            return False
        # 如果一个都放不下（单层超限），则该 mid 不可行
        if curCost == 0:
            return False
    return groups <= K

def solve():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    N = int(next(it))
    K = int(next(it))
    M = int(next(it))

    C = [int(next(it)) for _ in range(N)]
    W = [int(next(it)) for _ in range(N)]

    # 边界条件
    if K > N:
        sys.stdout.write("-1")
        return
    if any(w > M for w in W):
        sys.stdout.write("-1")
        return

    layers = list(zip(C, W))
    lo = max(C)  # 最小可能的最大耗时
    hi = sum(C)  # 最大可能的最大耗时

    ans = hi
    while lo <= hi:
        mid = (lo + hi) // 2
        if canPartition(layers, K, M, mid):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    sys.stdout.write(str(ans))

if __name__ == "__main__":
    solve()
```

## 第3题-流式日志Top-K高频统计（P4983）| 困难 | 2000ms

### 题目描述

在大数据实时监控系统中，服务器每秒产生海量日志数据。为了快速识别热点事件、异常流量或热门关键词，需要实现一个滑动窗口内的**流式 Top-K 高频词统计**组件。该组件需要高效处理高频词的添加、过期和查询。

请实现支持以下两种操作的功能：

**add(word)**：向滑动窗口中添加一个关键词。

1. 窗口大小为 W，当窗口内元素超过 W 时，自动淘汰最早进入窗口的元素（FIFO 规则）。
2. 被淘汰元素的频率计数需相应减少，若频率减为 0，则从统计中移除。

**get_top(k)**：返回当前窗口内频率最高的 k 个词。

1. 按频率降序排列，若频率相同，按字典序升序排列。
2. 若有效词不足 k 个，返回所有有效词。

### 输入描述

第 1 行：两个整数 `W Q`，用空格分隔。

- W：滑动窗口大小（1 ≤ W ≤ 100000）
- Q：操作总数（1 ≤ Q ≤ 100000）

接下来 Q 行，每行一个操作，格式为以下两种之一：

- `add x`：添加关键词操作。x 为关键词，由小写英文字母 [a-z] 组成，长度 [1, 20]
- `get k`：查询 Top-K 高频词操作。k 为需要查询的高频词数量（1 ≤ k ≤ 100000）

约束：
- 所有 get 操作执行时，滑动窗口内至少有一个元素（窗口不为空）。
- 每个测试用例中至少包含一次 get 操作。

### 输出描述

对于每个 `get k` 操作，输出一行结果。若结果包含多个关键词，关键词之间用单个空格分隔，行尾无多余空格。

### 样例1

**输入**
```
5 10
add banana
add apple
add cherry
add banana
add apple
get 3
add durian
add apple
add pipeapple
get 5
```

**输出**
```
apple banana cherry
apple banana durian pipeapple
```

**说明**

1. 操作 1-5：依次添加 banana, apple, cherry, banana, apple
   - 窗口满 5 个：`[banana, apple, cherry, banana, apple]`
   - 频率：banana=2, apple=2, cherry=1

2. 操作 6（get 3）：频率相同 banana 和 apple，按字典序 apple < banana
   - 输出：`apple banana cherry`

3. 操作 7-9：依次添加 durian, apple, pipeapple
   - add durian：淘汰 banana（FIFO），窗口 `[apple, cherry, banana, apple, durian]`，频率：apple=2, cherry=1, banana=1, durian=1
   - add apple：淘汰 cherry，窗口 `[cherry, banana, apple, durian, apple]`，频率：apple=2, banana=1, durian=1
   - add pipeapple：淘汰 banana，窗口 `[banana, apple, durian, apple, pipeapple]`，频率：apple=2, banana=1, durian=1, pipeapple=1

4. 操作 10（get 5）：有 4 个词，窗口不足 K，返回全部
   - 按频率降序 → 同频按字典序升序 → 输出：`apple banana durian pipeapple`

### 样例2

**输入**
```
3 8
add a
add b
add a
get 2
add b
add b
add b
get 2
```

**输出**
```
a b
b
```

**说明**

1. 操作 1-3：依次添加 a, b, a
   - 窗口满 3 个：`[a, b, a]`
   - 频率：a=2, b=1

2. 操作 4（get 2）：按频率降序 → 输出 `a b`

3. 操作 5-7：连续 add b 三次
   - 每次 evict 最早元素，最终窗口 `[b, b, b]`
   - 频率：b=3

4. 操作 8（get 2）：仅 1 个词，k=2，返回全部 → 输出 `b`

### 数据约束与算法提示

- 核心数据结构：
  - **队列（deque）**：维护窗口内元素的 FIFO 顺序
  - **哈希表（dict）**：维护每个词的频率计数
  - **有序集合（平衡树/堆 + 惰性删除）**：高效获取 Top-K
- W 和 Q 均可达 10^5，需要 O(log N) 级别的操作复杂度
- 被淘汰元素频率减为 0 时，需要从有序集合中移除

:::fold[答案]
## 解题思路
使用双端队列维护窗口 FIFO 顺序，哈希表维护词频。Top-K 查询采用堆 + 惰性删除：每次频率变更时向最小堆中压入 (-freq, word) 新条目，查询时不断弹堆顶，若弹出的频率与哈希表中当前频率不一致则丢弃（惰性删除），否则计入结果。由于同频率按字典序升序，( -freq, word ) 的元组排序天然满足要求。

## 参考代码
```python
import sys
import heapq
from collections import deque

def solve():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    first = data[0].split()
    W = int(first[0])
    Q = int(first[1])

    window = deque()       # 维护窗口 FIFO 顺序
    freq = {}              # 词 -> 当前频率
    heap = []              # 最小堆: (-freq, word)，惰性删除

    outLines = []

    for lineIdx in range(1, Q + 1):
        parts = data[lineIdx].split()
        op = parts[0]

        if op == "add":
            word = parts[1]
            window.append(word)
            freq[word] = freq.get(word, 0) + 1
            # 将新频率压入堆（惰性删除：旧条目留在堆中）
            heapq.heappush(heap, (-freq[word], word))

            # 窗口满则淘汰最早元素
            if len(window) > W:
                evict = window.popleft()
                freq[evict] -= 1
                if freq[evict] == 0:
                    del freq[evict]
                # 堆中旧条目由惰性删除处理，无需显式移除

        elif op == "get":
            k = int(parts[1])
            result = []
            seen = set()

            # 备份堆顶元素用于回推（惰性删除后需回推有效元素）
            temp = []
            while heap and len(result) < k:
                negFreq, word = heapq.heappop(heap)
                curFreq = freq.get(word, 0)
                if curFreq == 0 or word in seen:
                    # 惰性删除：频率归零或已加入结果
                    continue
                if curFreq != -negFreq:
                    # 频率不匹配，说明有更新的条目在堆中，丢弃
                    continue
                seen.add(word)
                result.append(word)
                temp.append((-negFreq, word))

            # 将取出的有效元素回推到堆中（供下次查询使用）
            for item in temp:
                heapq.heappush(heap, item)

            outLines.append(" ".join(result))

    sys.stdout.write("\n".join(outLines))

if __name__ == "__main__":
    solve()
```
