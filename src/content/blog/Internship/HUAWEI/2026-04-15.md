## 第2题-基于 AdamW 优化的网络带宽预测模型（P4779）| 中等 | 1000ms

### 题目描述

在华为网络通信业务中，网络带宽预测模型是保障数据传输稳定性的核心模块之一，通过历史数据拟合的带宽模型为：**y = w1 * x1 + w2 * x2 + b**（其中 y 表示带宽，x1 和 x2 为影响带宽的因子，w1 和 w2 为权重参数，b 为偏置参数）。请实现 **AdamW** 优化算法，基于给定样本数据迭代更新模型参数。

#### 核心概念

**损失函数**：对于单个样本 (x1, x2, y_true)，损失 L = (y_pred - y_true)^2，其中 y_pred = w1 * x1 + w2 * x2 + b。

**梯度计算**：
- g_w1 = dL/dw1 = 2 * (y_pred - y_true) * x1
- g_w2 = dL/dw2 = 2 * (y_pred - y_true) * x2
- g_b = dL/db = 2 * (y_pred - y_true)

**AdamW 算法参数**：
- 动量参数：β1 = 0.9，β2 = 0.999
- 权重衰减系数：λ = 0.01
- 学习率：α = 0.001
- 数值稳定性常数：ε = 1e-8

**一阶动量（m）更新**：m_t = β1 * m_{t-1} + (1 - β1) * g_t

**二阶动量（v）更新**：v_t = β2 * v_{t-1} + (1 - β2) * g_t^2

**偏差修正**：
- m̂_t = m_t / (1 - β1^t)
- v̂_t = v_t / (1 - β2^t)
（t 为当前迭代次数，从 1 开始）

**参数更新**：θ_t = θ_{t-1} - α * (m̂_t / (sqrt(v̂_t) + ε) + λ * θ_{t-1})
其中 θ 表示 (w1, w2, b)

**初始参数**：
- w1 = 0.0, w2 = 0.0, b = 0.0
- 初始一阶动量 m_w1 = 0.0, m_w2 = 0.0, m_b = 0.0
- 初始二阶动量 v_w1 = 0.0, v_w2 = 0.0, v_b = 0.0

### 输入格式

第一行输入一个整数 N，表示样本数量。

接下来 N 行，每行 3 个浮点数 x1, x2, y_true，表示一个样本。

### 输出格式

一行，3 个浮点数，依次为训练后的 w1, w2, b，结果保留 6 位小数，银行家舍入，以一个空格分隔，前后无冗余空格。

### 样例 1

**输入**
```
3
1.0 1.0 2.0
2.0 2.0 4.0
3.0 3.0 6.0
```

**输出**
```
0.002750 0.002750 0.002923
```

**说明**：样本数量为 3，迭代更新后输出最终模型参数。

### 样例 2

**输入**
```
1
0.0 0.0 0.0
```

**输出**
```
0.000000 0.000000 0.000000
```

**说明**：单个零样本，梯度始终为零，参数保持初始值。

:::fold[答案]

## 解题思路
遍历所有$N$个样本一次（单epoch），逐样本执行AdamW优化。每个样本先计算预测值和MSE损失梯度，然后更新一阶动量$m$和二阶动量$v$，做偏差修正后按AdamW规则更新$w_1$、$w_2$、$b$（含权重衰减项$\lambda \theta$）。注意平方梯度$g_t^2$即$(g_t)^2$，$v_t$更新时使用$g_t^2$而非$|g_t|^2$的歧义，严格按题目公式实现。输出保留$6$位小数。

## 参考代码
```python
import sys
import math


def solve():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    idx = 0
    N = int(data[idx]); idx += 1

    samples = []
    for _ in range(N):
        x1 = float(data[idx]); x2 = float(data[idx + 1]); yt = float(data[idx + 2])
        idx += 3
        samples.append((x1, x2, yt))

    # 初始参数
    w1, w2, b = 0.0, 0.0, 0.0
    mW1, mW2, mB = 0.0, 0.0, 0.0
    vW1, vW2, vB = 0.0, 0.0, 0.0

    beta1 = 0.9
    beta2 = 0.999
    lam = 0.01
    alpha = 0.001
    eps = 1e-8

    t = 0
    for x1, x2, yt in samples:
        t += 1

        # 前向预测
        yp = w1 * x1 + w2 * x2 + b
        err = yp - yt

        # 梯度
        gW1 = 2.0 * err * x1
        gW2 = 2.0 * err * x2
        gB = 2.0 * err

        # AdamW: w1
        mW1 = beta1 * mW1 + (1.0 - beta1) * gW1
        vW1 = beta2 * vW1 + (1.0 - beta2) * gW1 * gW1
        mHatW1 = mW1 / (1.0 - beta1 ** t)
        vHatW1 = vW1 / (1.0 - beta2 ** t)
        w1 -= alpha * (mHatW1 / (math.sqrt(vHatW1) + eps) + lam * w1)

        # AdamW: w2
        mW2 = beta1 * mW2 + (1.0 - beta1) * gW2
        vW2 = beta2 * vW2 + (1.0 - beta2) * gW2 * gW2
        mHatW2 = mW2 / (1.0 - beta1 ** t)
        vHatW2 = vW2 / (1.0 - beta2 ** t)
        w2 -= alpha * (mHatW2 / (math.sqrt(vHatW2) + eps) + lam * w2)

        # AdamW: b
        mB = beta1 * mB + (1.0 - beta1) * gB
        vB = beta2 * vB + (1.0 - beta2) * gB * gB
        mHatB = mB / (1.0 - beta1 ** t)
        vHatB = vB / (1.0 - beta2 ** t)
        b -= alpha * (mHatB / (math.sqrt(vHatB) + eps) + lam * b)

    print(f"{w1:.6f} {w2:.6f} {b:.6f}")


if __name__ == "__main__":
    solve()
```

## 第3题-大模型推理资源的最低成本分发（P4780）| 中等 | 1000ms

### 题目描述

当前只有若干个并发的大模型推理服务器，推理资源紧张，但是有 N 个推理请求任务在申请推理服务中，每个推理服务都有一个优先级的分值。要求对每个推理请求任务分发推理资源，每个任务至少分配 1 千个 token 消耗的推理资源。相邻的两个任务中，优先级越高的那个任务会获得更多的 token 数（X 千个 token 数）。请给每个推理请求任务分发 token，确保 N 个请求任务消耗的 token 总数最少（X 千个 token）。

如果优先级小于等于 0 或者为空，该任务需要被放弃，不分配任务 token，它的相邻任务不与该任务进行优先级比较，只和另一个比较。例如：[1, -1, 2] 的 token 总数是 2 千个。

### 输入格式

输入是一个优先级的数组，数值越大，优先级越高。例如：1, 2, 3。

### 输出格式

输出一个整数，表示最少 token 总数（千个）。

### 样例 1

**输入**
```
10,10,5
```

**输出**
```
4
```

**说明**：分别给第一、二、三个任务分发 1 千、2 千、1 千个 token。

### 样例 2

**输入**
```
4,2,6
```

**输出**
```
5
```

**说明**：分别给第一、二、三个任务分发 2 千、1 千、2 千个 token。

:::fold[答案]

## 解题思路
经典LeetCode 135"分发糖果"问题。先过滤掉优先级$\le 0$的任务（它们不分配token，也不参与相邻比较），对剩余有效任务：每项至少$1$个token，若当前任务优先级高于左邻居则token数$=$左邻居$+1$（从左到右遍历），若高于右邻居则token数$=\max($自身$, $右邻居$+1)$（从右到左遍历）。最后求和即为最少token总数。跳过无效任务时，其相邻有效任务成为新的相邻关系。

## 参考代码
```python
import sys


def solve():
    data = sys.stdin.read().strip()
    if not data:
        print(0)
        return

    # 解析输入：逗号分隔的优先级数组
    parts = [x.strip() for x in data.split(',')]
    priorities = []
    for p in parts:
        if p == '':
            priorities.append(0)  # 空值视为无效
        else:
            priorities.append(int(p))

    # 过滤出有效任务（优先级>0）
    valid = [(i, p) for i, p in enumerate(priorities) if p > 0]

    if not valid:
        print(0)
        return

    n = len(valid)
    tokens = [1] * n

    # 左->右：若当前优先级高于左侧，token数=左侧+1
    for i in range(1, n):
        if valid[i][1] > valid[i - 1][1]:
            tokens[i] = tokens[i - 1] + 1

    # 右->左：若当前优先级高于右侧，token数取max
    for i in range(n - 2, -1, -1):
        if valid[i][1] > valid[i + 1][1]:
            tokens[i] = max(tokens[i], tokens[i + 1] + 1)

    total = sum(tokens)
    print(total)


if __name__ == "__main__":
    solve()
```
