### 3. 在RLHF流程中，关于奖励模型的作用，最准确的是？
- A. 奖励模型是一个可微分的人类偏好代理，将人类对答案好坏的主观判断转化为可用于优化的连续可导奖励信号
- B. 奖励模型本身就是一个强化学习智能体
- C. 奖励模型的目标是学习模仿人类的写作风格
- D. 奖励模型直接修改基座模型的权重

:::fold[答案]

正确答案：**A**

> 延伸阅读：[[NeuralNetwork/LLM/RLHF|RLHF 算法详解：从 PPO 到 DPO 的对齐技术演进]]

**解析：**

本题考察 RLHF（基于人类反馈的强化学习）流程中奖励模型（Reward Model, RM）的核心定位与作用。正确理解奖励模型的角色是掌握 RLHF 三阶段流程的关键。

---

**1. 奖励模型在 RLHF 中的位置**

RLHF 的标准三阶段流程为：
1. **SFT（监督微调）**：用高质量对话数据训练基座模型
2. **RM（奖励模型训练）**：用人类偏好排序数据训练打分模型
3. **PPO（强化学习优化）**：以 RM 为奖励信号，用 PPO 算法优化语言模型

奖励模型位于阶段 2，充当阶段 3 的**奖励信号提供者**。它接收 Prompt + Response，输出一个标量分数 $r_\phi(x, y)$，分数越高表示回答越好。

---

**2. 逐项分析**

**A ✓ 奖励模型是一个可微分的人类偏好代理，将人类对答案好坏的主观判断转化为可用于优化的连续可导奖励信号**

这是对奖励模型最准确的定义，包含了三个关键要素：

**① 人类偏好代理（Proxy of Human Preferences）**

标注者不会给每个回答打绝对分数，而是进行**相对比较**——给定同一个 Prompt 的两个回答，选择"哪个更好"。这种数据称为偏好排序数据（preference ranking data）：

$$ \mathcal{D}_{\text{pref}} = \{(x, y_a, y_b) \mid \text{人类认为 } y_a \text{ 优于 } y_b\} $$

奖励模型通过学习这些比较数据，成为一个能够模仿人类判断的"代理"。

**② 可微分（Differentiable）**

奖励模型本质上是一个神经网络，其输出的标量分数对模型参数 $\phi$ 可导。在阶段 3 的 PPO 优化中，语言模型通过最大化奖励分数的梯度来更新参数：

$$ \nabla_\theta \, \mathbb{E}_{y \sim \pi_\theta}[r_\phi(x, y)] $$

如果奖励信号不可微分，就无法使用基于梯度的 RL 方法进行优化。

**③ 连续可导奖励信号（Continuous Differentiable Reward Signal）**

奖励模型将人类模糊的"好/坏"二元偏好转化为连续的标量信号，使得 RL 算法可以感知"好多少"或"差多少"的程度差异，从而进行精细的梯度优化。

**B ✗ 奖励模型本身就是一个强化学习智能体**

这是对 RLHF 架构的常见误解。区分三个角色的职责：

- **Agent（智能体）**：是阶段 3 中的**语言模型**（策略 $\pi_\theta$），它根据当前状态（Prompt + 已生成 Token）选择动作（下一个 Token）
- **Environment（环境）**：奖励模型和 KL 惩罚共同构成环境的一部分，为 Agent 的动作提供奖励反馈
- **Reward Model（奖励模型）**：仅是奖励信号的来源，不执行动作、不与环境交互、不维护策略

> 记忆技巧：在 RLHF 中，语言模型是"参赛选手"，奖励模型是"裁判"，PPO 算法是"教练"。

**C ✗ 奖励模型的目标是学习模仿人类的写作风格**

将两个概念混淆了：

- **学习模仿人类写作**是 SFT（阶段 1）的核心目标——通过人类撰写的高质量回答数据，让模型学会"像人一样写作"
- **学习判断回答好坏**是 RM（阶段 2）的核心目标——通过偏好比较数据，让模型学会"评判回答质量"

RLHF 的分工是分层的：SFT 关心"怎么写"，RM 关心"写得好不好"。

**D ✗ 奖励模型直接修改基座模型的权重**

奖励模型在训练完成后**冻结**（参数不再更新）。在阶段 3 的 PPO 训练中：

- 奖励模型仅用于**前向传播**：输入生成的回答，输出奖励分数
- **不参与反向传播**：奖励模型不接收来自 PPO 的梯度更新
- 真正更新权重的是**策略模型**（语言模型 $\pi_\theta$）和**价值模型**（Critic $V_\psi$）

---

**3. 小结**

| 说法 | 正误 | 核心要点 |
|:--|:--:|:--|
| 奖励模型是可微分的人类偏好代理 | ✅ | 准确描述了 RM 的本质：将主观偏好转化为可导信号 |
| 奖励模型是 RL 智能体 | ❌ | RM 不执行动作，智能体是语言模型本身 |
| RM 学习模仿人类写作 | ❌ | 那是 SFT 的任务，RM 学习的是"评判" |
| RM 直接修改基座权重 | ❌ | RM 在 PPO 阶段冻结，仅提供前向打分 |

核心直觉：**奖励模型 = 裁判**，它把人类"我觉得 A 比 B 好"的主观判断，转化为可以被梯度下降优化的数字信号。

:::

### 4. 关于先验概率和后验概率，正确的是
- A. 后验概率可以大于1
- B. 后验概率必须通过贝叶斯定理计算
- C. 先验概率是已知条件概率后的修正结果
- D. 先验概率是未考虑新信息前的初始概率

:::fold[答案]

正确答案：**D**

**解析：**

本题考察先验概率（prior probability）与后验概率（posterior probability）的基本概念，两者是贝叶斯统计中的核心术语。

---

**1. 概念辨析**

**先验概率** $P(H)$：在观测到任何数据或证据**之前**，基于已有知识或经验，对事件 $H$ 发生概率的初始估计。例如：在没有进行任何医学检测时，根据人群发病率估计某人患某病的概率。

**后验概率** $P(H \mid E)$：在观测到证据 $E$ **之后**，对事件 $H$ 发生概率的修正估计。例如：在拿到阳性检测报告后，重新评估患病的概率。

贝叶斯定理将两者联系起来：

$$
P(H \mid E) = \frac{P(E \mid H) \cdot P(H)}{P(E)}
$$

其中 $P(E \mid H)$ 为似然（likelihood），$P(E)$ 为边缘概率（evidence / marginal likelihood）。

---

**2. 逐项分析**

**A ✗ 后验概率可以大于 1**

概率的公理化定义要求任何概率值 $\in [0, 1]$，后验概率本质仍是概率，不可能大于 1。该说法违反概率基本公理。

> 反例：无论先验和似然如何取值，贝叶斯公式的分子 $P(E \mid H) \cdot P(H) \le P(E)$ 恒成立（因为 $P(E) = \sum_H P(E \mid H)P(H)$），故 $P(H \mid E) \le 1$ 严格成立。

**B ✗ 后验概率必须通过贝叶斯定理计算**

后验概率确实常通过贝叶斯定理求得，但并非"必须"。在某些场景下可以直接计算：

- 当联合分布已知时，可直接从条件概率定义出发：$P(H \mid E) = \frac{P(H, E)}{P(E)}$。
- 在频率学派视角下，后验也可以通过 MCMC 采样、变分推断等方法近似，不必显式写出贝叶斯公式。
- 简单的离散场景中，枚举所有可能性同样能算出后验。

贝叶斯定理是**工具**而非**唯一定义**。

**C ✗ 先验概率是已知条件概率后的修正结果**

这个描述恰恰把先后关系说反了。"已知条件概率后修正"得到的应该是**后验概率**，而非先验概率。先验概率是在获得新信息**之前**就确定的，是修正的起点，不是修正的结果。

> 记忆技巧："先"验 = 先于证据，"后"验 = 后于证据。

**D ✓ 先验概率是未考虑新信息前的初始概率**

这是先验概率的标准定义。在观测任何新数据之前，仅凭历史经验或主观判断给出的概率值即为先验概率。当新数据到来后，通过贝叶斯更新将其修正为后验概率。

---

**3. 小结**

| 概念 | 时机 | 符号 | 含义 |
|------|------|------|------|
| 先验概率 | 观测数据**前** | $P(H)$ | 对假设的初始信念 |
| 后验概率 | 观测数据**后** | $P(H \mid E)$ | 综合证据后的修正信念 |

核心直觉：**先验 + 数据 → 后验**。贝叶斯定理就是这条更新路径的数学表达。
:::

### 6. MDP中Q-learning更新：$s_1 \to a_1 \to s_2$（终态），$r=3$，$\gamma=0.9$，$\alpha=0.1$，初始 $Q=0$。更新后 $Q(s_1,a_1)=$？
- A. $0.4$
- B. $0.3$
- C. $0.2$
- D. $0.5$

:::fold[答案]

正确答案：**B**

> 详细讲解见 [[MachineLearning/ReinforcementLearning/Qlearning|Q-learning 算法详解]]。

**解析：**

本题考察 Q-learning 更新公式的直接代入计算，是强化学习中时序差分（TD）方法的基础题型。

---

**1. Q-learning 更新公式**

Q-learning 的核心是以下 TD 更新规则：

$$
\boxed{Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \cdot \max_{a'} Q(s', a') - Q(s, a) \right]}
$$

各项含义：

| 符号 | 含义 | 本题取值 |
|:--|:--|:--|
| $Q(s, a)$ | 状态 $s$ 下执行动作 $a$ 的当前价值估计 | 初始 $Q = 0$ |
| $\alpha$ | 学习率，控制每次更新的步长 | $0.1$ |
| $r$ | 执行动作 $a$ 后获得的即时奖励 | $3$ |
| $\gamma$ | 折扣因子，权衡短期与长期回报 | $0.9$ |
| $s'$ | 执行动作 $a$ 后转移到的下一状态 | $s_2$（终态） |
| $\max_{a'} Q(s', a')$ | 下一状态所有可能动作的最大 Q 值 | $0$（终态无可选动作） |

---

**2. TD 误差分解**

括号内的表达式 $r + \gamma \max_{a'} Q(s', a') - Q(s, a)$ 称为 **TD 误差（Temporal Difference Error）**，由两部分构成：

- **TD 目标（target）**：$r + \gamma \max_{a'} Q(s', a')$ — 基于实际奖励和下一状态价值估计的"更准确的"回报估计
- **当前估计（current）**：$Q(s, a)$ — 更新前的价值估计

Q-learning 的思想是：让当前估计 $Q(s, a)$ 向 TD 目标**移动一小步**（步长 $\alpha$）。

---

**3. 逐项代入**

本题中 $s_2$ 是终态，没有后续动作可选，因此 $\max_{a'} Q(s_2, a') = 0$：

$$
\begin{aligned}
Q_{\text{new}}(s_1, a_1) &= Q_{\text{old}}(s_1, a_1) + \alpha \left[ r + \gamma \cdot 0 - Q_{\text{old}}(s_1, a_1) \right] \\
&= 0 + 0.1 \times (3 + 0 - 0) \\
&= 0.1 \times 3 \\
&= 0.3
\end{aligned}
$$

---

**4. 各选项验证**

- **A. $0.4$**：可能是 $\alpha r = 0.1 \times 3 = 0.3$，但误加了额外项。也可能是用 $r \times (\gamma + \alpha)$ 等错误公式
- **B. $0.3$** ✓：正确代入的结果
- **C. $0.2$**：可能是 $\alpha r - 0.1$ 或误用 $\alpha(r - \gamma)$
- **D. $0.5$**：可能是 $\alpha r / (1 - \gamma) \times 0.1$ 的错误思路

---

**5. 关键要点**

- 当下一状态为**终态**时，$\max_{a'} Q(s', a') = 0$，更新退化为 $Q \leftarrow Q + \alpha(r - Q)$
- 由于初始 $Q = 0$，第一次更新等价于 $Q = \alpha r$（TD 目标 = $r$）
- $\gamma$ 在终态场景中不参与计算，这是常见陷阱——题目给出 $\gamma = 0.9$ 但不影响结果
:::

### 7. 在高斯混合模型（GMM）中，EM算法的E步主要计算：
- A. 参数更新
- B. 后验概率
- C. 特征函数
- D. 先验概率

:::fold[答案]

正确答案：**B**

**解析：**

本题考察高斯混合模型（GMM）中 EM 算法的核心机制，属于经典机器学习基础题型。关键在于区分 E 步（Expectation）和 M 步（Maximization）各自的职责。

---

**1. GMM 与 EM 算法概述**

**高斯混合模型（Gaussian Mixture Model, GMM）** 假设数据由 $K$ 个高斯分布混合生成：

$$
p(\mathbf{x}) = \sum_{k=1}^{K} \pi_k \cdot \mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)
$$

其中 $\pi_k$ 是第 $k$ 个高斯分量的混合系数（先验概率），$\boldsymbol{\mu}_k$ 是均值向量，$\boldsymbol{\Sigma}_k$ 是协方差矩阵。

问题在于：如果直接对似然 $p(\mathbf{X} \mid \boldsymbol{\theta})$ 求导，会得到耦合的非线性方程组，没有闭式解。**EM 算法**通过引入隐变量来解耦。

**EM 算法的核心思想**：迭代执行两步直到收敛——

| 步骤 | 全称 | 做什么 | 核心输出 |
|:--|:--|:--|:--|
| **E 步** | Expectation（期望） | 基于当前参数 $\boldsymbol{\theta}^{(t)}$，计算隐变量的后验分布 | 责任值 $\gamma_{ik}$ |
| **M 步** | Maximization（最大化） | 固定责任值，最大化完整数据的期望对数似然 | 新参数 $\boldsymbol{\theta}^{(t+1)}$ |

---

**2. E 步的数学细节（本题核心）**

在 GMM 中，隐变量 $z_i \in \{1, \dots, K\}$ 表示第 $i$ 个样本来自哪个高斯分量。E 步计算的是：

$$
\boxed{\gamma_{ik} = P(z_i = k \mid \mathbf{x}_i, \boldsymbol{\theta}^{(t)}) = \frac{\pi_k \cdot \mathcal{N}(\mathbf{x}_i \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)}{\sum_{j=1}^{K} \pi_j \cdot \mathcal{N}(\mathbf{x}_i \mid \boldsymbol{\mu}_j, \boldsymbol{\Sigma}_j)}}
$$

这就是**后验概率**（posterior probability），也称为**责任值**（responsibility）。分母是所有 $K$ 个分量的加权概率密度之和，确保 $\sum_{k=1}^K \gamma_{ik} = 1$。

**直觉理解**：
- $\gamma_{ik}$ 量化了"在当前的参数估计下，第 $i$ 个样本有多大可能来自第 $k$ 个高斯分量"
- 它是一个「软分配」（soft assignment）——不是硬性地把样本划给某个分量，而是给出每个分量的概率
- 这个后验概率将在 M 步中作为「权重」，用于加权更新均值和协方差

**为什么叫「期望」？** E 步本质是求隐变量 $z_i$ 在观测 $\mathbf{x}_i$ 条件下的条件期望：
$$
\mathbb{E}_{z \mid \mathbf{x}, \boldsymbol{\theta}^{(t)}}[\log p(\mathbf{x}, z \mid \boldsymbol{\theta})]
$$
而 $\gamma_{ik} = \mathbb{E}[z_{ik} \mid \mathbf{x}_i]$ 正是这个期望中的核心量。

---

**3. 逐项分析**

**A ✗ 参数更新**

参数更新（计算新的 $\boldsymbol{\mu}_k$、$\boldsymbol{\Sigma}_k$、$\pi_k$）是 **M 步**的工作。M 步利用 E 步算出的责任值 $\gamma_{ik}$，通过加权最大似然估计来更新参数：

$$
\boldsymbol{\mu}_k^{(t+1)} = \frac{\sum_{i=1}^{N} \gamma_{ik} \mathbf{x}_i}{\sum_{i=1}^{N} \gamma_{ik}}, \quad
\boldsymbol{\Sigma}_k^{(t+1)} = \frac{\sum_{i=1}^{N} \gamma_{ik} (\mathbf{x}_i - \boldsymbol{\mu}_k^{(t+1)})(\mathbf{x}_i - \boldsymbol{\mu}_k^{(t+1)})^\top}{\sum_{i=1}^{N} \gamma_{ik}}
$$

$$
\pi_k^{(t+1)} = \frac{1}{N} \sum_{i=1}^{N} \gamma_{ik}
$$

将 A 选项和 B 选项混淆是笔试中常见的错误——需要牢记 **E 步 = 求后验**，**M 步 = 更新参数**。

**B ✓ 后验概率**

如第 2 节所述，E 步的输出就是样本属于各分量的后验概率（责任值）。这是 EM 算法的「半个心跳」——E 步承担了将隐变量「猜测」出来的角色。

**C ✗ 特征函数**

特征函数（characteristic function）是概率论中描述随机变量分布的工具，定义为 $\varphi_X(t) = \mathbb{E}[e^{itX}]$，与 EM 算法和 GMM 没有直接关系。该选项属于干扰项。

**D ✗ 先验概率**

先验概率 $P(z_i = k)$ 在没有观测数据时表示样本属于各分量的初始信念。在 GMM 中，先验就是混合系数 $\pi_k$。后验概率 $P(z_i = k \mid \mathbf{x}_i)$ 才是在观测到数据后的修正信念，这也是 E 步实际计算的对象。

> 本题与第 4 题（先验/后验概率概念）形成呼应：E 步本质上是**贝叶斯更新**——用观测数据 $\mathbf{x}_i$ 将先验 $\pi_k$ 更新为后验 $\gamma_{ik}$。

---

**4. EM 算法完整流程（伪代码）**

```
初始化: π_k, μ_k, Σ_k (k = 1, ..., K)

重复直到收敛:
    # === E 步：计算后验概率 ===
    对每个样本 i 和每个分量 k:
        γ_ik = π_k * N(x_i | μ_k, Σ_k) / sum_j[π_j * N(x_i | μ_j, Σ_j)]

    # === M 步：最大化期望对数似然 ===
    对每个分量 k:
        N_k = sum_i γ_ik                    # 有效样本数
        μ_k = (1/N_k) * sum_i γ_ik * x_i    # 加权均值
        Σ_k = (1/N_k) * sum_i γ_ik * (x_i - μ_k)(x_i - μ_k)^T  # 加权协方差
        π_k = N_k / N                        # 混合系数
```

---

**5. 小结**

| 选项 | 正误 | 归属步骤 | 核心要点 |
|:--|:--:|:--|:--|
| 参数更新 | ❌ | M 步 | E 步不动参数，只算责任值 |
| **后验概率** | ✅ | **E 步** | 计算每个样本属于各分量的条件概率 |
| 特征函数 | ❌ | 无关 | 概率论工具，与 EM/GMM 无关 |
| 先验概率 | ❌ | 无关 | E 步计算的是后验，非先验 |

**记忆口诀**：**E 步求后验，M 步更新参**。E = Expectation = 期望 = 计算隐变量的后验分布；M = Maximization = 最大化 = 用该后验加权求解参数。

## 第2题-大模型训练显存优化算法（P4625）| 中等 | 2000ms

### 题目描述

在大模型训练中，为解决 $NPU$ 显存不足的问题，通常采用张量 $swap$ 或者张量重计算的方式进行优化；张量 $swap$ 是把张量的数据先搬到 $CPU$，需要的时候，再从 $CPU$ 搬回 $NPU$；张量重计算是在需要张量的值时，在 $NPU$ 上重新把张量的值计算出来。

假设把模型部署在 $NPU$ 上，还需要 $m$ 大小的存储空间，才能让大模型运行起来；目前有 $n$ 个候选张量，每个张量占用一定的存储空间；$n$ 个张量中的每个张量都可以进行 $swap$ 或者重计算，$swap$ 和重计算分别对应不同的代价。

试着编写一段程序，从 $n$ 个候选张量中选择合适的张量进行 $swap$ 或者重计算，在代价最小的情况下，使得大模型能够运行起来（选择的张量和大于等于 $m$）。

### 输入格式

- 第一行为 $1$ 个整数 $m(0<m<10000)$，表示需要的存储空间大小
- 第二行为 $1$ 个整数 $n(0<n<10000)$，表示候选张量的个数
- 第三行有 $n$ 个处于区间 $[1, 100000]$ 之内的整数，表示 $n$ 个候选张量的存储空间大小
- 第四行有 $n$ 个处于区间 $[1, 100000]$ 之内的整数，表示 $n$ 个候选张量 $swap$ 的代价
- 第五行有 $n$ 个处于区间 $[1, 100000]$ 之内的整数，表示 $n$ 个候选张量重计算的代价

### 输出格式

如果没有找到合适的解，输出 $error$；否则，输出最小代价（行尾没有空格）。

### 样例1

**输入：**
```
10
5
3 4 5 6 7
1 2 3 5 5
2 3 4 5 6
```

**输出：**
```
6
```

**说明：**

需要的存储空间是 $10$；候选张量长度分别为 $3, 4, 5, 6, 7$；张量长度组合 $3+7, 4+6, 5+6, 5+7, 6+7, 3+4+5, ...$ 都大于等于 $10$，满足运行的基本条件；对于 $3+7$ 来说，$3$ 的最小代价是 $1$，$7$ 的最小代价是 $5$，$3+7$ 的最小代价是 $6$；经过其他组合的比较，达到目标的最小代价是 $6$，所以输出 $6$。

### 样例2

**输入：**
```
12
4
18 20 10 12
12 3 13 4
10 14 10 1
```

**输出：**
```
1
```

**说明：**

需要的存储空间是 $12$；候选张量长度分别为 $18, 20, 10, 12$；张量长度 $18, 20, 12$ 都大于等于 $12$，满足运行的基本条件；其中 $12$ 的代价最小为 $1$，所以输出 $1$。

:::fold[答案]

## 解题思路
每个张量可以选择swap或重计算，取两者中代价较小的操作。问题转化为0/1背包变种：从$n$个物品（张量）中选择一个子集，使得总空间$\ge m$且总代价最小。使用DP，$\text{dp}[j]$表示释放$j$空间的最小代价，$j$的范围为$[0, m]$，超过$m$的部分合并到$\text{dp}[m]$。遍历每个张量时从大到小更新$\text{dp}$，避免重复选择。最终$\text{dp}[m]$即为答案，若不可达则输出error。

## 参考代码
```python
import sys


def solve():
    data = sys.stdin.read().split()
    if not data:
        return
    idx = 0
    m = int(data[idx]); idx += 1
    n = int(data[idx]); idx += 1

    sizes = [int(data[idx + i]) for i in range(n)]
    idx += n
    swapCosts = [int(data[idx + i]) for i in range(n)]
    idx += n
    recCosts = [int(data[idx + i]) for i in range(n)]

    # 每个张量取swap和重计算中的最小代价
    items = []
    for i in range(n):
        cost = min(swapCosts[i], recCosts[i])
        items.append((sizes[i], cost))

    INF = 10 ** 18
    dp = [INF] * (m + 1)
    dp[0] = 0

    # 0/1背包DP，容量上限为m（超过m合并到dp[m]）
    for size, cost in items:
        for j in range(m, -1, -1):
            if dp[j] < INF:
                nj = j + size
                if nj >= m:
                    if dp[j] + cost < dp[m]:
                        dp[m] = dp[j] + cost
                else:
                    if dp[j] + cost < dp[nj]:
                        dp[nj] = dp[j] + cost

    if dp[m] == INF:
        print("error")
    else:
        print(dp[m])


if __name__ == "__main__":
    solve()
```

## 第3题-基于KNN的语音数据分类（P4626）| 中等 | 1000ms

### 题目描述

在终端穿戴场景中，往往涉及到用户和终端设备之间的语音交互，通常由用户唤醒相关的设备并进行语音命令控制设备完成相应的操作，设备在收到语音命令时会对用户的意图进行识别，例如识别"拍照""摄像"等意图，从而进行对应的操作。

要求使用 $KNN$ 算法对用户输出的语音特征进行分类：

语音片段经过特征提取后是一个 $3$ 维的向量，所有距离使用欧式距离公式计算，即对于两个 $N$ 维向量：

$$
\vec{x} = (x_1, x_2, \dots, x_N), \quad \vec{y} = (y_1, y_2, \dots, y_N)
$$

两个向量间的距离为：

$$
d_{(\vec{x},\vec{y})} = \sqrt{\sum_{k=1}^N (x_k - y_k)^2}
$$

### 输入格式

- 第一行：正整数 $N$ 和 $K$，分别表示给定的已知语音特征向量的数量和邻居数量 $K$
- 第 $2$ ~ $N+1$ 行：已知语音的特征向量和 $label$，其中最后一位表示类别 $label$
- 第 $N+2$ 行：待分类的语音特征向量

### 输出格式

给出待分类语音的类别，该值为正整数在输入给定范围内。

### 样例1

**输入：**
```
10 3
0.5 0.3 0.4 0
0.6 0.2 0.5 0
0.4 0.3 0.3 0
0.7 0.4 0.6 0
2.1 2.3 2.2 1
2.3 2.2 2.4 1
2.2 2.4 2.3 1
4.5 4.3 4.4 2
4.4 4.5 4.6 2
4.6 4.4 4.5 2
2.2 2.1 2.3
```

**输出：**
```
1
```

**说明：**

待分类的语音特征向量为 $(2.2, 2.1, 2.3)$。计算其与所有已知向量的欧式距离后，最近的 $3$ 个邻居分别是：

- $(2.1, 2.3, 2.2)$ 类别 $1$
- $(2.3, 2.2, 2.4)$ 类别 $1$
- $(2.2, 2.4, 2.3)$ 类别 $1$

这 $3$ 个邻居都属于类别 $1$，因此待分类向量被判定为类别 $1$。

### 样例2

**输入：**
```
12 4
1 0.9 1 1
0.8 0.9 0.7 1
1.3 1 1.2 1
1.2 0.9 1 1
2 2.2 2.1 2
2.3 2.2 2 2
2 2.2 1.9 2
1.9 2.2 2.1 2
3.1 3.1 3 3
2.8 2.9 3.1 3
2.9 3 3.2 3
3.1 3 3.1 3
2.2 1.2 1.9
```

**输出：**
```
2
```

**说明：**

与 $(2.2, 1.2, 1.9)$ 最近的 $4$ 个邻居中，类别最多的是 $2$，故被分类为 $2$。

:::fold[答案]

## 解题思路
标准KNN分类算法。计算待分类向量与所有已知样本的欧式距离，选取距离最小的$K$个邻居，统计$K$个邻居中出现次数最多的类别作为预测结果。若多个类别出现次数相同，取标签值最小的类别（按题目默认约定）。时间复杂度$O(N \cdot D + N \log N)$，其中$D=3$为特征维度。

## 参考代码
```python
import sys
import math


def solve():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    idx = 0
    N = int(data[idx]); idx += 1
    K = int(data[idx]); idx += 1

    # 读取已知样本
    samples = []
    for _ in range(N):
        x = float(data[idx]); y = float(data[idx + 1])
        z = float(data[idx + 2]); label = int(data[idx + 3])
        idx += 4
        samples.append((x, y, z, label))

    # 读取待分类向量
    qx = float(data[idx]); qy = float(data[idx + 1]); qz = float(data[idx + 2])

    # 计算所有距离
    distances = []
    for x, y, z, label in samples:
        d = math.sqrt((qx - x) ** 2 + (qy - y) ** 2 + (qz - z) ** 2)
        distances.append((d, label))

    # 按距离升序排序，取前K个
    distances.sort(key=lambda t: t[0])
    topK = distances[:K]

    # 统计K个邻居中各类别出现次数
    count = {}
    for _, label in topK:
        count[label] = count.get(label, 0) + 1

    # 找出现次数最多的类别，平局时取标签值较小者
    bestLabel = -1
    bestCount = -1
    for label, cnt in count.items():
        if cnt > bestCount or (cnt == bestCount and label < bestLabel):
            bestCount = cnt
            bestLabel = label

    print(bestLabel)


if __name__ == "__main__":
    solve()
```
:::
