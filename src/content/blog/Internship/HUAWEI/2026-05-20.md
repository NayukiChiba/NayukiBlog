## 第1题-选择题（P4962）共20道，1-15单选，16-20多选

### 1. 用二次多项式拟合数据点 $(0,1),(1,2),(2,3),(3,4)$，一次项系数最接近?
- A. 2
- B. 1
- C. 4
- D. 3

:::fold[答案]

正确答案：B

解析：四点恰好共线于 $y = x + 1$，二次项系数为 0，一次项系数为 1，截距为 1。最小二乘拟合结果与线性拟合一致。
:::

### 2. 双线性插值与最近邻插值相比主要可以减少?
- A. 计算时间
- B. 精度损失
- C. 视觉锯齿
- D. 数据存储

:::fold[答案]

正确答案：C

解析：双线性插值利用周围 4 个像素进行加权平均，输出图像更平滑，能有效减少最近邻插值产生的锯齿状边缘（aliasing/锯齿效应）。
:::

### 3. 多智能体联合状态向量组 $s_1=(1,3,5)^T$, $s_2=(2,4,6)^T$, $s_3=(3,5,7)^T$ 构成矩阵 $S$，则 $S$ 的零空间维度为（）
- A. 3
- B. 0
- C. 1
- D. 2

:::fold[答案]

正确答案：C

解析：$s_2 = s_1 + (1,1,1)^T$，$s_3 = s_1 + 2(1,1,1)^T$，三个向量张成的空间秩为 2（都在 $(1,3,5)$ 和 $(1,1,1)$ 的二维平面内），零空间维度 $=3-2=1$。
:::

### 4. 预条件共轭梯度法（PCG）中预条件矩阵 $M$ 的选取目标是使 $M^{-1}A$ 的条件数:
- A. 无所谓
- B. 尽可能接近1
- C. 等于A的条件数
- D. 尽可能大

:::fold[答案]

正确答案：B

解析：预条件矩阵 $M$ 的目标是使 $M^{-1}A$ 的条件数尽可能接近 1，从而加速共轭梯度法的收敛。条件数越接近 1，矩阵越接近单位矩阵，迭代收敛越快。
:::

### 5. $L_2$ 正则化系数 $\lambda$，正确的是:
- A. λ=0时正则化效果最强
- B. λ选择与过拟合/欠拟合无关
- C. λ越大模型越倾向拟合训练数据
- D. λ越大对权重惩罚越重，模型复杂度越低

:::fold[答案]

正确答案：D

解析：$L_2$ 正则化在损失函数中添加 $\lambda \sum w_i^2$ 惩罚项。$\lambda$ 越大，对权重的惩罚越重，权重越接近 0，模型复杂度越低（防止过拟合）。$\lambda=0$ 时无正则化效果。
:::

### 6. PagedAttention的核心设计目标是解决（）
- A. 重计算导致推理延迟高
- B. 显存碎片化导致利用率低
- C. 量化后精度损失
- D. KV Cache占用显存过大

:::fold[答案]

正确答案：B

解析：PagedAttention（vLLM 核心创新）借鉴操作系统分页思想管理 KV Cache，将 KV Cache 划分为固定大小的 Block，核心目标是解决显存碎片化导致的利用率低问题，同时实现近乎零浪费的 KV Cache 共享。
:::

### 7. 大模型安全对齐中"对齐失效"最常见的风险场景是?
- A. 推理速度过慢
- B. 词汇表过小
- C. 损失函数无法收敛
- D. 面对恶意prompt绕过安全限制生成有害内容

:::fold[答案]

正确答案：D

### 8. 弦截法（割线法）的收敛阶约为:
- A. 2阶
- B. 3阶
- C. 1阶
- D. 1.618阶

:::fold[答案]

正确答案：D

解析：弦截法（割线法）的收敛阶为黄金比例 $\phi = \frac{1+\sqrt{5}}{2} \approx 1.618$，介于线性收敛（1 阶）和二次收敛（2 阶，牛顿法）之间，是一种超线性收敛方法。
:::

### 9. 关于 KV Cache 在自回归推理中的作用，最准确的是:
- A. 减少参数量
- B. 提升训练收敛速度
- C. 缓存历史token的K/V，避免每步重复计算
- D. 让模型推理时不再需要注意力机制

:::fold[答案]

正确答案：C

解析：KV Cache 的核心作用是缓存历史 token 的 Key 和 Value 向量，避免在每一步自回归生成时重复计算所有历史 token 的 K 和 V，从而大幅减少计算量。
:::

### 10. LLM推理中prefill和decoding阶段的计算瓶颈，正确的是:
- A. 两个阶段都是compute-bound
- B. prefill是memory-bound，decoding是compute-bound
- C. prefill并行处理整个输入序列，矩阵乘法密集，是compute-bound；decoding每步生成一个token需反复加载全部权重，是memory-bound
- D. KV Cache使decoding变为compute-bound

:::fold[答案]

正确答案：C

:::

## 第2题-LLM多源语料分级清洗预算分配（P4963）| 困难 | 1000ms

### 题目描述

某 LLM 预训练团队从 N 个数据源收集语料，每个数据源 i 具有以下属性：

- **w_i**：占总训练 token 的权重比例，满足 Σ w_i = 1
- **r_i**：初始脏数据比例，0 < r_i < 1
- **c_i**：基础清洗单价（正整数）
- **t_i**：基础清洗算力消耗

对每个数据源，可独立选择以下四种清洗级别之一：

| 清洗级别 | 花费成本 | 清洗后脏数据比例 |
|---------|---------|----------------|
| 0（不清洗） | 0 | r_i |
| 1（粗清洗） | 1 × c_i | r_i × 0.6 |
| 2（精清洗） | 3 × c_i | r_i × 0.2 |
| 3（全清洗） | 6 × c_i | 0 |

整体语料的加权污染率定义为：

**R = Σ w_i × r_i'**

（其中 r_i' 为第 i 个数据源清洗后的脏数据比例）

在总花费不超过预算 B、总算力不超过 T 的前提下，为每个数据源选择一个清洗级别，使得加权污染率 R 最小。返回可达到的最小加权污染率，结果保留 **6 位小数**。

### 输入描述

第 1 行：`N B T`

- N：数据源数量
- B：总预算上限
- T：总算力上限

接下来 N 行，每行：`w_i r_i c_i t_i`

- w_i：第 i 个数据源的 token 权重比例，所有 w_i 之和为 1
- r_i：第 i 个数据源的初始脏数据比例
- c_i：第 i 个数据源的基础清洗单价
- t_i：第 i 个数据源的基础清洗算力消耗

### 输出描述

输出一个字符串，表示在预算 B 和算力 T 限制下可达到的最小加权污染率。结果保留 6 位小数，不包含多余空格或换行。

### 样例1

**输入**
```
3 10 7
0.4 0.5 2 2
0.3 0.7 3 3
0.3 0.4 1 1
```

**输出**
```
0.270000
```

**说明**

考虑三种方案：

- 源1选2级 + 源2选1级：花费 6+3=9，算力 4+3=7，源3不清洗 → R = 0.040 + 0.126 + 0.120 = **0.286**
- 源1选2级 + 源3选2级：花费 6+3=9，算力 4+2=6，源2不清洗 → R = 0.040 + 0.210 + 0.024 = **0.274**
- 源1选1级 + 源2选1级 + 源3选2级：花费 2+3+3=8，算力 2+3+2=7 → R = 0.120 + 0.126 + 0.024 = **0.270**

最小为 0.270000。

### 样例2

**输入**
```
2 6 4
0.5 0.8 2 2
0.5 0.6 3 1
```

**输出**
```
0.380000
```

**说明**

| 方案 | 源1级别 | 源2级别 | 花费 | 算力 | R |
|-----|--------|--------|------|------|---|
| A | 0（不清洗） | 0（不清洗） | 0 | 0 | 0.5×0.8+0.5×0.6 = 0.700 |
| B | 1（粗清洗） | 1（粗清洗） | 2+3=5 | 2+1=3 | 0.5×0.48+0.5×0.36 = 0.420 |
| C | 2（精清洗） | 0（不清洗） | 6 | 4 | 0.5×0.16+0.5×0.60 = 0.380 |
| D | 1（粗清洗） | 0（不清洗） | 2 | 2 | 0.5×0.48+0.5×0.60 = 0.540 |

最小为方案 C（源1选2级），R = 0.380000。

### 数据约束与算法提示

- 核心算法：**二维分组背包 DP**（预算 + 算力两个维度约束）
- 每个数据源为一组，组内 4 种清洗级别对应不同(花费, 算力, 污染贡献)
- 目标：在总花费 ≤ B 且总算力 ≤ T 下，最小化加权污染率 R

:::fold[答案]
## 解题思路
二维分组背包 DP。每个数据源为一组，组内 4 种清洗级别对应不同的（花费，算力，污染贡献）。用字典维护可达的 (花费, 算力) 状态及其最小污染率，逐源转移。最终取所有花费 ≤ B 且算力 ≤ T 的状态中的最小污染率。使用稀疏 DP 避免 O(N*B*T) 的密集数组开销。

## 参考代码
```python
import sys

def solve():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    N = int(next(it))
    B = int(next(it))
    T = int(next(it))

    sources = []
    for _ in range(N):
        w = float(next(it))
        r = float(next(it))
        c = int(next(it))
        t_ = int(next(it))
        sources.append((w, r, c, t_))

    # 每种清洗级别的 (花费乘数, 脏数据比例乘数)
    levels = [(0, 1.0), (1, 0.6), (3, 0.2), (6, 0.0)]

    # dp: (b, t) -> min pollution
    dp = {(0, 0): 0.0}

    for w, r, c, tcost in sources:
        newDp = {}
        for (bUsed, tUsed), curPol in dp.items():
            for costMul, dirtyMul in levels:
                nb = bUsed + costMul * c
                nt = tUsed + costMul * tcost
                if nb > B or nt > T:
                    continue
                newPol = curPol + w * r * dirtyMul
                key = (nb, nt)
                if key not in newDp or newPol < newDp[key]:
                    newDp[key] = newPol
        dp = newDp

    ans = min(dp.values()) if dp else 0.0
    sys.stdout.write(f"{ans:.6f}")

if __name__ == "__main__":
    solve()
```
:::

## 第3题-敏感实体动态遮蔽掩码（P4964）| 困难 | 2000ms

### 题目描述

为了防止大语言模型记忆并泄露输入上下文中的敏感数据，安全框架会对输入长文本进行预扫描，匹配预设的敏感词库（如 API_KEY、身份证号码等）。

一旦在序列中发现了敏感词，系统会在底层的注意力掩码矩阵中对这些"敏感区块"进行动态隔离遮蔽。具体规则：**敏感区块内部的词可以互相看到，但外部的任何词都绝对不能看到这些敏感数据。**

给定一个长度为 N 的 Token 序列 T，以及 K 个敏感模式串，处理流程如下：

**第一步：模式匹配与敏感区块合并**

1. 在序列 T 中找出所有敏感模式串的出现位置，得到一组闭区间 {[L1,R1], [L2,R2], ..., [Lm,Rm]}
2. 若两个区间满足 **相交** 或 **首尾相邻** 条件，则合并为一个更大的敏感块：
   - 相交：Ia ∩ Ib ≠ ∅
   - 首尾相邻：Lb = Ra+1 或 La = Rb+1

**第二步：动态遮蔽掩码计算**

对于任意 Token i 和目标 Token j（遵循因果律 j ≤ i），Token i 能否观察到 Token j，由以下规则决定：

- **隔离规则**：如果 j 属于某个敏感块，那么只有当 i 也身处同一个敏感块内部时，i 才能观察到 j
- **正常规则**：如果 j 不属于任何敏感块，它对所有满足 i ≥ j 的后续 Token 均正常可见

### 输入描述

第 1 行：两个整数 N 和 K（主序列长度、敏感模式串数量）

第 2 行：N 个整数（主序列的 Token ID）

接下来 K 行：每行首先是一个整数 M（敏感模式串长度），后跟 M 个整数（敏感模式串的 Token ID）

约束条件：
- 1 ≤ N ≤ 10^5
- 1 ≤ K ≤ 100
- 1 ≤ M ≤ 1000

### 输出描述

N 个整数，以空格分隔，代表每个位置 i 的 Token 在序列中总共能观察到多少个 Token。

### 样例1

**输入**
```
7 2
10 20 30 40 50 60 70
3 20 30 40
2 40 50
```

**输出**
```

```

**说明**

1. 模式匹配与敏感区块合并：
   - 敏感模式 [20,30,40] 在序列中的索引为 [1, 3]
   - 敏感模式 [40,50] 在序列中的索引为 [3, 4]
   - 两者首尾相邻（L2=R1+1），合并为一个大敏感块，索引 [1, 4]

2. 计算每个位置可见 Token 数量：
   - i=0（值10，正常）: 只能看到自己 → 1
   - i=1（值20，敏感块）: 前面正常的(1个) + 当前敏感块内(2个) → 2
   - i=2（值30，敏感块）: 前面正常的(1个) + 当前敏感块内(3个) → 3
   - i=3（值40，敏感块）: 前面正常的(1个) + 当前敏感块内(4个) → 4
   - i=4（值50，敏感块）: 前面正常的(1个) + 当前敏感块内(5个) → 5

### 样例2

**输入**
```
9 1
10 20 30 40 50 60 30 40 70
2 30 40
```

**输出**
```
1 2 3 4 3 4 5 6 5
```

**说明**

1. 模式匹配与敏感区块合并：
   - 敏感模式 [30,40] 在序列中出现两次：索引 [2,3] 和 [6,7]
   - 两次出现不相邻，形成两个独立的敏感块

2. 计算每个位置可见 Token 数量：
   - i=0（值10，正常）: 1
   - i=1（值20，正常）: 2
   - i=2（值30，敏感块1）: 前面正常的(2个) + 敏感块内(2个) → 3
   - i=3（值40，敏感块1）: 前面正常的(2个) + 敏感块内(2个) → 4
   - i=4（值50，正常）: 前面正常的(3个) → 3
   - i=5（值60，正常）: 前面正常的(4个) → 4
   - i=6（值30，敏感块2）: 前面正常的(4个) + 敏感块内(2个) → 5
   - i=7（值40，敏感块2）: 前面正常的(4个) + 敏感块内(2个) → 6
   - i=8（值70，正常）: 前面正常的(5个) → 5

### 数据约束与算法提示

- 核心算法：**KMP 多模式匹配 + 区间合并（排序后贪心合并）+ 前缀和/前缀计数**
- 计算每个位置可见 token 数时：若当前位置 i 在敏感块 [L,R] 内，可见 token 数为「[1,L-1] 内正常 token 数 + 当前位置到敏感块起始的距离 + 1」
- 注意题目要求输出的是**每个位置能看到的 token 总数**（包括自身），而非未遮蔽 token 数

:::fold[答案]
## 解题思路
先用 KMP 对每个模式串在主序列中匹配出所有出现区间，按起始位置排序后合并相交或相邻区间。构建前缀数组 prefixNormal[i] 表示 [0, i] 内非敏感位置的个数。对位置 i：若不在敏感块内则答案为 prefixNormal[i]；若在敏感块 [L,R] 内则答案为 prefixNormal[i] + (i - L + 1)。即正常 token 可见所有前面的正常 token，敏感 token 额外可见同块内前面的敏感 token。

## 参考代码
```python
import sys

def computeKMP(pattern, seq):
    """KMP 返回 pattern 在 seq 中所有匹配的起始索引（含重叠）"""
    m = len(pattern)
    if m == 0:
        return []
    # 构建 next 数组
    nxt = [0] * m
    j = 0
    for i in range(1, m):
        while j > 0 and pattern[i] != pattern[j]:
            j = nxt[j - 1]
        if pattern[i] == pattern[j]:
            j += 1
        nxt[i] = j

    res = []
    j = 0
    for i, ch in enumerate(seq):
        while j > 0 and ch != pattern[j]:
            j = nxt[j - 1]
        if ch == pattern[j]:
            j += 1
        if j == m:
            res.append(i - m + 1)
            j = nxt[j - 1]  # 允许重叠匹配
    return res

def solve():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    first = data[0].split()
    N = int(first[0])
    K = int(first[1])
    seq = list(map(int, data[1].split()))

    intervals = []
    lineIdx = 2
    for _ in range(K):
        parts = list(map(int, data[lineIdx].split()))
        lineIdx += 1
        M = parts[0]
        pattern = parts[1:]  # 长度为 M
        starts = computeKMP(pattern, seq)
        for s in starts:
            e = s + M - 1
            intervals.append((s, e))

    if not intervals:
        # 没有敏感块，每个 token 可见前面所有 token（含自身）
        ans = [str(i + 1) for i in range(N)]
        sys.stdout.write(" ".join(ans))
        return

    intervals.sort()
    merged = [intervals[0]]
    for s, e in intervals[1:]:
        lastS, lastE = merged[-1]
        if s <= lastE + 1:  # 相交或相邻
            merged[-1] = (lastS, max(lastE, e))
        else:
            merged.append((s, e))

    # prefixNormal[i] = [0..i] 中不在任何敏感块内的元素个数
    prefixNormal = [0] * N
    # 标记敏感块区间用于快速判断
    blockStart = [-1] * N  # 该位置所属敏感块的起始下标，-1 表示非敏感
    for s, e in merged:
        for idx in range(s, e + 1):
            blockStart[idx] = s

    cnt = 0
    for i in range(N):
        if blockStart[i] == -1:
            cnt += 1
        prefixNormal[i] = cnt

    ans = []
    for i in range(N):
        L = blockStart[i]
        if L == -1:
            ans.append(str(prefixNormal[i]))
        else:
            total = prefixNormal[i] + (i - L + 1)
            ans.append(str(total))

    sys.stdout.write(" ".join(ans))

if __name__ == "__main__":
    solve()
```
