## 第2题-LLM多源语料分级清洗预算分配（P4963）| 困难 | 1000ms

### 题目描述

某 LLM 预训练团队从 N 个数据源收集语料，每个数据源 i 具有以下属性：

- **w_i**：占总训练 token 的权重比例，满足 Σ w_i = 1
- **r_i**：初始脏数据比例，0 < r_i < 1
- **c_i**：基础清洗单价（正整数）
- **t_i**：基础清洗算力消耗

对每个数据源，可独立选择以下四种清洗级别之一：

| 清洗级别 | 花费成本 | 清洗后脏数据比例 |
|---------|---------|----------------|
| 0（不清洗） | 0 | r_i |
| 1（粗清洗） | 1 × c_i | r_i × 0.6 |
| 2（精清洗） | 3 × c_i | r_i × 0.2 |
| 3（全清洗） | 6 × c_i | 0 |

整体语料的加权污染率定义为：

**R = Σ w_i × r_i'**

（其中 r_i' 为第 i 个数据源清洗后的脏数据比例）

在总花费不超过预算 B、总算力不超过 T 的前提下，为每个数据源选择一个清洗级别，使得加权污染率 R 最小。返回可达到的最小加权污染率，结果保留 **6 位小数**。

### 输入描述

第 1 行：`N B T`

- N：数据源数量
- B：总预算上限
- T：总算力上限

接下来 N 行，每行：`w_i r_i c_i t_i`

- w_i：第 i 个数据源的 token 权重比例，所有 w_i 之和为 1
- r_i：第 i 个数据源的初始脏数据比例
- c_i：第 i 个数据源的基础清洗单价
- t_i：第 i 个数据源的基础清洗算力消耗

### 输出描述

输出一个字符串，表示在预算 B 和算力 T 限制下可达到的最小加权污染率。结果保留 6 位小数，不包含多余空格或换行。

### 样例1

**输入**
```
3 10 7
0.4 0.5 2 2
0.3 0.7 3 3
0.3 0.4 1 1
```

**输出**
```
0.270000
```

**说明**

考虑三种方案：

- 源1选2级 + 源2选1级：花费 6+3=9，算力 4+3=7，源3不清洗 → R = 0.040 + 0.126 + 0.120 = **0.286**
- 源1选2级 + 源3选2级：花费 6+3=9，算力 4+2=6，源2不清洗 → R = 0.040 + 0.210 + 0.024 = **0.274**
- 源1选1级 + 源2选1级 + 源3选2级：花费 2+3+3=8，算力 2+3+2=7 → R = 0.120 + 0.126 + 0.024 = **0.270**

最小为 0.270000。

### 样例2

**输入**
```
2 6 4
0.5 0.8 2 2
0.5 0.6 3 1
```

**输出**
```
0.380000
```

**说明**

| 方案 | 源1级别 | 源2级别 | 花费 | 算力 | R |
|-----|--------|--------|------|------|---|
| A | 0（不清洗） | 0（不清洗） | 0 | 0 | 0.5×0.8+0.5×0.6 = 0.700 |
| B | 1（粗清洗） | 1（粗清洗） | 2+3=5 | 2+1=3 | 0.5×0.48+0.5×0.36 = 0.420 |
| C | 2（精清洗） | 0（不清洗） | 6 | 4 | 0.5×0.16+0.5×0.60 = 0.380 |
| D | 1（粗清洗） | 0（不清洗） | 2 | 2 | 0.5×0.48+0.5×0.60 = 0.540 |

最小为方案 C（源1选2级），R = 0.380000。

### 数据约束与算法提示

- 核心算法：**二维分组背包 DP**（预算 + 算力两个维度约束）
- 每个数据源为一组，组内 4 种清洗级别对应不同(花费, 算力, 污染贡献)
- 目标：在总花费 ≤ B 且总算力 ≤ T 下，最小化加权污染率 R

## 第3题-敏感实体动态遮蔽掩码（P4964）| 困难 | 2000ms

### 题目描述

为了防止大语言模型记忆并泄露输入上下文中的敏感数据，安全框架会对输入长文本进行预扫描，匹配预设的敏感词库（如 API_KEY、身份证号码等）。

一旦在序列中发现了敏感词，系统会在底层的注意力掩码矩阵中对这些"敏感区块"进行动态隔离遮蔽。具体规则：**敏感区块内部的词可以互相看到，但外部的任何词都绝对不能看到这些敏感数据。**

给定一个长度为 N 的 Token 序列 T，以及 K 个敏感模式串，处理流程如下：

**第一步：模式匹配与敏感区块合并**

1. 在序列 T 中找出所有敏感模式串的出现位置，得到一组闭区间 {[L1,R1], [L2,R2], ..., [Lm,Rm]}
2. 若两个区间满足 **相交** 或 **首尾相邻** 条件，则合并为一个更大的敏感块：
   - 相交：Ia ∩ Ib ≠ ∅
   - 首尾相邻：Lb = Ra+1 或 La = Rb+1

**第二步：动态遮蔽掩码计算**

对于任意 Token i 和目标 Token j（遵循因果律 j ≤ i），Token i 能否观察到 Token j，由以下规则决定：

- **隔离规则**：如果 j 属于某个敏感块，那么只有当 i 也身处同一个敏感块内部时，i 才能观察到 j
- **正常规则**：如果 j 不属于任何敏感块，它对所有满足 i ≥ j 的后续 Token 均正常可见

### 输入描述

第 1 行：两个整数 N 和 K（主序列长度、敏感模式串数量）

第 2 行：N 个整数（主序列的 Token ID）

接下来 K 行：每行首先是一个整数 M（敏感模式串长度），后跟 M 个整数（敏感模式串的 Token ID）

约束条件：
- 1 ≤ N ≤ 10^5
- 1 ≤ K ≤ 100
- 1 ≤ M ≤ 1000

### 输出描述

N 个整数，以空格分隔，代表每个位置 i 的 Token 在序列中总共能观察到多少个 Token。

### 样例1

**输入**
```
7 2
10 20 30 40 50 60 70
3 20 30 40
2 40 50
```

**输出**
```
1 2 3 4 5 2 3
```

**说明**

1. 模式匹配与敏感区块合并：
   - 敏感模式 [20,30,40] 在序列中的索引为 [1, 3]
   - 敏感模式 [40,50] 在序列中的索引为 [3, 4]
   - 两者首尾相邻（L2=R1+1），合并为一个大敏感块，索引 [1, 4]

2. 计算每个位置可见 Token 数量：
   - i=0（值10，正常）: 只能看到自己 → 1
   - i=1（值20，敏感块）: 前面正常的(1个) + 当前敏感块内(2个) → 2
   - i=2（值30，敏感块）: 前面正常的(1个) + 当前敏感块内(3个) → 3
   - i=3（值40，敏感块）: 前面正常的(1个) + 当前敏感块内(4个) → 4
   - i=4（值50，敏感块）: 前面正常的(1个) + 当前敏感块内(5个) → 5
   - i=5（值60，正常）: 前面正常的(2个) → 2
   - i=6（值70，正常）: 前面正常的(3个) → 3

### 样例2

**输入**
```
9 1
10 20 30 40 50 60 30 40 70
2 30 40
```

**输出**
```
1 2 3 4 3 4 5 6 5
```

**说明**

1. 模式匹配与敏感区块合并：
   - 敏感模式 [30,40] 在序列中出现两次：索引 [2,3] 和 [6,7]
   - 两次出现不相邻，形成两个独立的敏感块

2. 计算每个位置可见 Token 数量：
   - i=0（值10，正常）: 1
   - i=1（值20，正常）: 2
   - i=2（值30，敏感块1）: 前面正常的(2个) + 敏感块内(2个) → 3
   - i=3（值40，敏感块1）: 前面正常的(2个) + 敏感块内(2个) → 4
   - i=4（值50，正常）: 前面正常的(3个) → 3
   - i=5（值60，正常）: 前面正常的(4个) → 4
   - i=6（值30，敏感块2）: 前面正常的(4个) + 敏感块内(2个) → 5
   - i=7（值40，敏感块2）: 前面正常的(4个) + 敏感块内(2个) → 6
   - i=8（值70，正常）: 前面正常的(5个) → 5

### 数据约束与算法提示

- 核心算法：**KMP 多模式匹配 + 区间合并（排序后贪心合并）+ 前缀和/前缀计数**
- 计算每个位置可见 token 数时：若当前位置 i 在敏感块 [L,R] 内，可见 token 数为「[1,L-1] 内正常 token 数 + 当前位置到敏感块起始的距离 + 1」
- 注意题目要求输出的是**每个位置能看到的 token 总数**（包括自身），而非未遮蔽 token 数
