GitHub 上常见的开源许可证在**使用条件、版权声明、专利授权、兼容性**等方面有显著区别。以下是主流许可证的核心差异及总结表格：

---

# 常见开源许可证区别

1. **MIT**
   - **最宽松**，只需保留版权声明和许可证文本
   - 允许商用、修改、分发、私用，无专利明确授权
   - 兼容几乎所有其他许可证

2. **Apache 2.0**
   - 类似 MIT，但**明确授予专利授权**，并提供专利诉讼保护条款
   - 要求修改文件时需注明更改
   - 兼容 GPLv3，但不兼容 GPLv2

3. **GPL（GNU General Public License）**
   - **Copyleft 强传染性**：任何衍生作品（即使修改很小）也必须以相同许可证开源
   - 分发的二进制文件需提供源码
   - **GPLv3** 额外禁止“Tivoization”（硬件锁定）和明确专利授权

4. **LGPL（GNU Lesser GPL）**
   - 较 GPL 宽松，允许以**动态链接**方式在闭源软件中使用库
   - 仅修改库本身时才需开源

5. **BSD（2-Clause/3-Clause）**
   - **BSD-2**：类似 MIT，仅需保留版权声明
   - **BSD-3**：增加“不得用作者名推广衍生作品”条款

6. **AGPL（Affero GPL）**
   - **最严格 Copyleft**：即使通过网络提供服务（SaaS），也必须开源修改后的代码

---

# 许可证对比总结表

| 许可证        | 核心要求           | 允许商用 | 修改/分发要求      | 专利授权 | 兼容性          |
| ---------- | -------------- | ---- | ------------ | ---- | ------------ |
| MIT        | 保留版权和许可证声明     | ✅    | 可闭源分发        | ❌    | 极高           |
| Apache 2.0 | 保留版权、变更说明、专利声明 | ✅    | 可闭源分发        | ✅    | 高（不兼容 GPLv2） |
| GPLv3      | 衍生作品必须同许可证开源   | ✅    | 必须开源且提供源码    | ✅    | 仅兼容同 GPL 系列  |
| LGPL       | 动态链接可闭源，修改库需开源 | ✅    | 修改库部分需开源     | ❌    | 较高           |
| BSD-3      | 保留版权、不得用作者名推广  | ✅    | 可闭源分发        | ❌    | 极高           |
| AGPL       | 网络服务也必须开源修改代码  | ✅    | 必须开源（含 SaaS） | ✅    | 严格（仅兼容 AGPL） |

---

# 如何选择？
- **个人/小项目、希望广泛使用** → MIT / BSD
- **库/工具、重视专利保护** → Apache 2.0
- **希望衍生作品保持开源** → GPL（软件）/ LGPL（库）
- **网络服务项目、坚持强开源** → AGPL
